Understanding and determining phrase-set syntax-summary

"Phrase-set syntax" is when a large set of phrases is expressed in a single line (or few lines). For example:

    zTPFindFlag[MC,RE,WW][on,off]

states the following six phrases are available:

  • zTPFindFlagMCon
  • zTPFindFlagMCoff
  • zTPFindFlagREon
  • zTPFindFlagREoff
  • zTPFindFlagWWon
  • zTPFindFlagWWoff

This document describes how to understand phrase-set syntax-summary, and also how to take a large set of phrases and create its syntax-summary (including determining the number of phrases in a set, based on its syntax).

[go to determining phrase-count]

"Phrase-set syntax" is when a large set of phrases is expressed in a single line (or few lines). For example:

    zTPFindFlag[MC,RE,WW][on,off]

states the following six phrases are available:

  • zTPFindFlagMCon
  • zTPFindFlagMCoff
  • zTPFindFlagREon
  • zTPFindFlagREoff
  • zTPFindFlagWWon
  • zTPFindFlagWWoff

The single square-bracket
    [MC,RE,WW]
means that "exactly one of these elements is required". If it were this instead (and it's not!)
    zTPFindFlag[[MC,RE,WW]][on,off]
it would mean these three options are optional, and therefore two additional phrases are available:

  • zTPFindFlagon
  • zTPFindFlagoff

    zFindDlgClose[[Down,Up]][MC][[RE,WW]][XS]

When a single item is in a single square bracket, like
    '[MC]' or '[XS]'
that item is optional. When two or more items are in a single square bracket, like
    '[MC,RE,WW]'
then that item is required. When two or more items are in two square brackets, like
    '[[MC,RE,WW]]'
then it is optional. (When it is indeed used, it must be one of the comma-separated values).

[BACK]   Determining the number of phrases in a phrase-set, based on its syntax-summary

To determine the amount of available phrases given its syntax, create a multiplication formula, where there are [total-option-blocks] numbers, and each number is the total amount of possibilities in that option-block. Using the above example,
    zFindDlgClose[[Down,Up]][MC][[RE,WW]][XS]
has four total option blocks, implying
    ? * ? * ? * ?

Option-block one
    [[Down,Up]]
has three options:

  1. nothing
  2. 'Down'
  3. 'Up'

    3 * ? * ? * ?

Option-block two
    [MC]
has two options:

  1. nothing
  2. 'MC'

    3 * 2 * ? * ?

Option-block three
    [[RE,WW]]
has three options:

  1. nothing
  2. 'RE'
  3. 'WW'

    3 * 2 * 3 * ?

Option-block four
    [XS]
has two options:

  1. nothing
  2. 'XS'

    3 * 2 * 3 * 2

And therefore there are
    3 * 2 * 3 * 2
    6 * 6
    36
phrases.

Here is the concise syntax for these functions:

fkg[v][c] -- Func in THIS class
fkgdb[v][c][k] -- Func in diff-class, diff-dir (file in cb, dir prompt)
fkgf[b][v][c][k] -- Func in diff-class, same-dir (dir='')
fkgp[f[b],r][k] -- fkg, no post-</A> params
fkgr[v][c][k] -- Func in super-class (dir-prompt)

OPTIONS:
[v]=Parameters should be "post-</A>" abbreviated
[c]=Surround the link with "<CODE>...</CODE>"
[k]=Include a class-link-dot, immediately previous to the link-proper
[f[b]]=Create link with file-name [b]=in clipboard, [not b]=user-prompt

While you still may not understand the specifics of these functions and their options, users already familiar with the basics can clearly understand every possible option at a glance.
