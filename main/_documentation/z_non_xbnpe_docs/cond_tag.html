PhraseExpress COND tag: Documentation

[go to notes]

The {#COND} command outputs text, depending on a condition. Specifically, if its '-if' condition is

  • met: text in the '-then' clause is output
  • not met: text in the '-else' clause is output
The real power of a {#COND} command is in deciding the name of a phrase to be executed.

All examples in this document are intended to be called from within a text document. Also note that they are displayed in uncompressed form, with extra line-breaks and indentation for clarity. Here is the phrase-file containing all these examples.

The simplest example, where {#COND} simply outputs text:

cond01simple
{#COND -if 3 < 1 
   -then 3 LT 1 
   -else 3 GTOE 1}

Execute 'cond01simple' and this is output:
    3 GTOE 1

(Note that both

cond02fail1
{#COND -if 3 < 1 
   -then 3 <= 1 
   -else 3 LT 1}

and

cond03fail2
{#COND -if 3 < 1 
   -then 3 <= 1 
   -else 3 < 1}

fail--they do not execute at all.)

(A reminder that these extra line-breaks are for visual clarity only. They are illegal in actuality.)

Importantly, ALL phrases in the '-then' and '-else' clauses are executed--regardless the condition. While, executing 'cond04callins' results in the same output (note that symbols are still not okay, even in called phrases)...

cond04callins
{#COND -if 3 < 1 
   -then {#insert z3lt1} 
   -else {#insert z3nlt1}}
z3lt1
3 LT 1
z3nlt1
3 GTOE 1

...it is not ideal, since the all phrases in BOTH the '-then' and '-else' clauses are executed--the output of the '-then' clause is suppressed, but it is STILL executed. To demonstrate this, execute 'cond05callinsdbgs':

cond05callinsdbgs
{#COND -if 3 < 1 
   -then {#insert z3lt1WDbg} 
   -else {#insert z3nlt1WDbg}}
z3lt1WDbg
3 LT 1{#MSGBOX -head DEBUG: z3lt1WDbg -text Currently in z3lt1WDbg}
z3nlt1WDbg
3 GTOE 1{#MSGBOX -head DEBUG: z3nlt1WDbg -text Currently in z3nlt1WDbg}

As expected, only this is printed
    3 GTOE 1
but both dialogs appear, and before the text is output.

The real power of the {#COND} command comes in determining which phrase to execute. For example, 

cond06inins
{#insert {#COND -if 3 < 1 
   -then z3lt1 
   -else z3nlt1}}

Which results in
    {#insert z3nlt1}
and therefore:
    3 GTOE 1
Since only the phrase-name is generated by the {#COND} command, nothing extra is executed.

To conclude with a slightly-more realistic example:

cond07namelen
{#insert {#COND -if {#LENGTH {#insert zYourNamePrompt}} <= 5 
   -then zNameIsLTOE5Chars 
   -else zNameIsGT5Chars}}
zYourNamePrompt
{#INPUT -head Your name? -def Humphrey -single}
zNameIsGT5Chars
Your name, {#insert zYourNamePrompt}, has more than five characters.
zNameIsLTOE5Chars
Your name, {#insert zYourNamePrompt}, has exactly (or less-than) five characters.

Call 'cond07namelen', and this is output:
    Your name, Humphrey, has more than five characters.

[top]   COND tag: Notes and observations

If any of the following unescaped characters are in your '-if' condition, the COND tag will fail: =, !, <, >.

CHAR    ASC#
---------------------
=       {#asc 61}
!       {#asc 33}
<       {#asc 60}
>       {#asc 62}

Here is a working example with escaped symbols in the '-if' condition:

cond08escif
{#COND -if \\\<hello\\\> = \\\<hello\\\> 
   -then EQUAL 
   -else not-equal}

(12/12/2012: http://www.phraseexpress.com/phpBB3/viewtopic.php?f=138&t=10690)

Character comparison example

Executing

cond09chars
{#COND -if a < 0 -then a LT 0 -else a GTOE 0}{#ENTER}
{#COND -if a > 9 -then a GT 9 -else a LTOE 9}{#ENTER}
{#COND -if + < 0 -then + LT 0 -else + GTOE 0}{#ENTER}
{#COND -if + > 9 -then + GT 9 -else + LTOE 9}{#ENTER}
{#COND -if {#asc 32} < 0 -then space LT 0 -else space GTOE 0}{#ENTER}
{#COND -if {#asc 32} > 9 -then space GT 9 -else space LTOE 9}{#ENTER}
{#COND -if z < 0 -then z LT 0 -else z GTOE 0}{#ENTER}
{#COND -if z > 9 -then z GT 9 -else z LTOE 9}{#ENTER}
{#COND -if @ < 0 -then @ LT 0 -else @ GTOE 0}{#ENTER}
{#COND -if @ > 9 -then @ GT 9 -else @ LTOE 9}

results in

a GTOE 0
a LTOE 9
+ GTOE 0
+ LTOE 9
space GTOE 0
space LTOE 9
z GTOE 0
z LTOE 9
@ GTOE 0
@ LTOE 9

Comments on either side of the '-if' parameter

cond10ifcmt
{#COND -if yes == yes(*a test comment*) 
   -then YES 
   -else NO}

and no spaces around the comparison operator

cond11nospaceif1
{#COND -if yes==yes 
   -then YES 
   -else NO}

Works in some situations, and doesn't in others. It is strongly recommended that you avoid comments in the '-if' clause, and always have spaces around the comparator.

Example

cond12nospaceif2
{#COND -if off != off 
   -then not equal 
   -else yes equal}{#ENTER}
{#COND -if off!=off 
   -then not equal 
   -else yes equal}
yes equal
not equal

An interesting example

This won't work:

cond132balloons1{#COND -if {#INPUT -head Trigger word [1/3]? (it's 'trigger')} -single} = trigger 
   -then {#BALLOON -title Right word -text You have typed the correct word.} 
   -else {#BALLOON -title Wrong word -text You haven't typed the correct word.}}

but this will:

cond142balloons2
{#insert {#COND -if {#input -head Trigger word [2/3]? (it's 'trigger')} = trigger 
   -then zRightBalloon 
   -else zWrongBalloon}}
zRightBalloon
{#BALLOON -title Right word -text You have typed the correct word.}
zWrongBalloon
{#BALLOON -title Wrong word -text You haven't typed the correct word.}

as will this:

cond152balloons3
{#SETPHRASE -description zvTRIGGER_TEXT -content {#input -head Trigger word [3/3]? (it's 'trigger')} -autotext zvTRIGGER_TEXT -folder ctvv_folder}
{#BALLOON -title {#COND -if {#insert zvTRIGGER_TEXT} == trigger 
   -then Right 
   -else Wrong} word -text You have{#COND -if {#insert zvTRIGGER_TEXT} != trigger 
      -then n't 
      -else} typed the correct word}

I strongly prefer the former, as it clearly separates logic and presentation.

Note to those familiar with C++ and Java, and the like:

cond162equals
{#COND -if on == on 
   -then ON! 
   -else OFF!}{#ENTER}
{#COND -if on = on 
   -then ON! 
   -else OFF!}

Output:

OFF!
ON!

Comparing against the empty-string

cond17emptystring
{#COND -if x = {#insert zzEmptyString} 
   -then 'x' IS the empty string?! 
   -else 'x' is not the empty-string}{#ENTER}
{#COND -if y =  -then 'y' IS the empty string?! -else 'y' is not the empty-string}{#ENTER}
{#COND -if  =  -then '' is the empty string -else '' is NOT the empty-string?!}

Output:

'x' is not the empty-string
'y' is not the empty-string
'' is the empty string
