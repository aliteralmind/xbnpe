TPKC: Master configuration variables

These critical settings customize the behavior of the TextPad built-in command (TPKC) project. They are collectively called the TPKC "Master Configuration Variables", or tpkc-master-cfg-vars.

[top]   TPKC master-cfg-var: zForceTestKeyCmd

The key-command used to "force" a TPKC nbs-command test. Not unlike TextPad-bootstraps, this also has a required corresponding "escaped" version:
    zForceTestKeyCmdEsc

For example, if zForceTestKeyCmd is equal to
    {#insert zASpOO}
then zForceTestKeyCmdEsc must be equal to
    \\\{#insert zASpOO\\\}

Requirements

  • This specific key-command (sequence of key-presses, independent of sleep) may not be used by any command, as configured in the TPKC-project. Specifically, it may not be equal to the tp-bootstrap command, nor any of it nbs-kcs.
  • It is strongly-recommended that each key-press be an x-key-press.
  • When this key-command has multiple key-presses. Each key-press should be separated by zTpkcMillsBtwMultiKeyKCs milliseconds of sleep.
  • It is not necessarily required to have sleep at they very end of this phrase, since it is always called in isolation (a message-box is popped up after each TPKC-nbs-test).

[top]   TPKC master-cfg-var: zTpkcAllCategoryAbrvLCsQbl_FORTESTMODE

Requirements: Must be an exact subset of zTpkcAllCategoryAbrvLCsQbl

The sQuare-Bracketed-List defining the universe of TextPad command categories, used only when building the TPKC-project in "test mode". Specifically, when zTpkcBuildModeTestOrLive is "live", then the universe of TextPad categories is defined instead by
    zTpkcAllCategoryAbrvLCsQbl
When zTpkcBuildModeTestOrLive is "test", the universe is defined by this master-cfg-var:
    zTpkcAllCategoryAbrvLCsQbl_FORTESTMODE

As of TextPad version 7.0.9, the value of
    zTpkcAllCategoryAbrvLCsQbl
is
    [file][edit][search][view][tools][macros][config][window][help][cursor][sel][cliplib][chars]

zTpkcAllCategoryAbrvLCsQbl_FORTESTMODE must be an exact subset of zTpkcAllCategoryAbrvLCsQbl. For example:
    [file][config][help]

The purpose of this test-set of categories is for testing and demonstrating the TPKC-build process.

[top]   TPKC master-cfg-var: zTpkcBuildModeTestOrLive

Requirements: Must be equal to "live" or "test"

Determines whether the current TPKC-build is in "live" or "test" mode. When equal to "live", the universe of TextPad command categories is defined by
    zTpkcAllCategoryAbrvLCsQbl
When "test", the universe is instead defined by the TPKC master-configuration-variable
    zTpkcAllCategoryAbrvLCsQbl_FORTESTMODE

[top]   TPKC master-cfg-var: zTpkcMaxNbsKCsPerCmd

Requirements: Must be an integer greater than zero.

The maximum number of key-commands that any one built-in TextPad command may have, after all tp-bootstrap key-commands have been deleted. For more information, see "The maximum nbs-kc count" under TextPad all-command-list.

A larger number will make the TPKC-build process slower. I personally set this value to 5.

[top]   TPKC master-cfg-var: zTpkcMillsBtwMultiKeyKCs

Requirement: Must be an integer greater than zero (a number valid for {#sleep ...}).

The number of milliseconds to sleep between multi-key key commands. For example, if your TextPad all-command-list has this command-line:
    FileCloseAll            Ctrl+F12, Ctrl+F9
and zTpkcMillsBtwMultiKeyKCs is equal to 25, then the TPKC-build-process translates it into the nbs-phrase

zTPFileCloseAll
   {#insert zCpF12}{#sleep 25}{#insert zCpF9}

The phrase-name for the command FileCloseAll is configured to be zTPFileCloseAll.
