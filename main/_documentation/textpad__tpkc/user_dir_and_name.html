TPKC user-directory and user-dir-name

The TPKC user-directory is where all your custom settings are stored. Before it is usable, it must be assigned a name.

Note that, aside from your user-dir-name, these structures are already setup for you.

[top]   TPKC-user-dir: PhraseExpress folder structure

The structure of the TPKC user-directory, as it exists in the PhraseExpress application:

   XBNPE: Parent folder >
      XBNPE: TextPad built-in commands (TPKC) >
         XBNPE-TPKC: User-spcfc phrases: [YOUR_TPKC_USER_DIR_NAME] >
            Nbs TP-phrases [CALL THESE!] (AG:14) & has-a-kc qbl-s     (info)
            User-specific command configuration (AG:13)               (info)
            z-DDMENU of all active TP-built-ins (AG:15)               (info)

[BACK]   TPKC user-dir: ...\output\usr_spcfc_cfg__bldproc10thr13\file.px.txt

There is one phrase-file in this directory, for every category that

  • has at least one command,
  • whose nbs-kc-count is greater than zero
  • (whether or not it is a TextPad-bootstrap).

As a whole, these phrase-files are known as your "user-specific command-configuration" (or user-cmd-cfg). Whenever you change an nbs-kc in
    TextPad > Configure > Preferences > Keyboard
that change must be reflected in its user-cmd-cfg category-file. This is required in order for the assign-kc process to assign key-commands as you expect.

If the primary-kc is changed (see "terminology"), then TPKC sub-build-processes-14 must be re-run. However, when it is known that only one-or-a-few categories have been changed, it is possible to execute build-proc-14 with category prompts.

If a command that never had a key-command now does (or you just deleted the final key-command from a no-longer-needed command), then it is strongly-recommended that sub-build-proc-15 also be re-run. This is in order to ensure the xh... phrase-creator ddmenu is accurate.

(When changing a TextPad-bootstrap, its main, escaped, and tp-key phrases must all be changed, in phrase-files unrelated to user-cmd-cfg.)

[BACK]   TPKC user-dir: ...\output\nbs_command_phrases__bldproc14\file.px.txt

There is one phrase-file in this directory, for every category that

  • has at least one command,
  • whose nbs-kc-count is greater than zero, and
  • is not a TextPad-bootstrap.

These phrases are the final output of the TPKC project. When any phrase (outside of TPKC and XBN-PhraseExpress) need to trigger a specific TextPad built-in command, these nbs-phrases are the ones they should call. The only exceptions are TextPad-bootstraps, which are not included in these phrase-files.

These phrase-files must be imported into

   XBNPE: Parent folder >
      XBNPE: TextPad built-in commands (TPKC) >
         XBNPE-TPKC: User-spcfc phrases: [YOUR_TPKC_USER_DIR_NAME] >
            Nbs TP-phrases [CALL THESE!] (AG: 14) & has-a-kc qbl-s

[BACK]   TPKC user-dir: ...\output\tp_all_active_cmds__bldproc15__pcddmenu.px.txt

This phrase-file serves as the foundation to the xh... phrase-creator ddmenu. This file is equal to
    tp_all_active_cmds__pcddmenu__indented_DO_NOT_IMPORT.txt
except that it is completely unindented (and is therefore importable)

This phrase-file must be imported into

   XBNPE: Parent folder >
      XBNPE: TextPad built-in commands (TPKC) >
         XBNPE-TPKC: User-spcfc phrases: [YOUR_TPKC_USER_DIR_NAME] >
            z-DDMENU of all active TP-built-ins (AG:15)

[BACK]   TPKC user-dir: ...\output\XBNPE-TPKC- User-specific phrases- [YOUR_TPKC_USER_DIR_NAME]

This is the backup file of your PhraseExpress TPKC-user-folder. You are encouraged to back this folder up (by exporting, via right-clicking on the folder in PhraseExpress, and typing Ctrl+E) to this location after any changes are made (after any phrase-files are imported, as described elsewhere in this document).
