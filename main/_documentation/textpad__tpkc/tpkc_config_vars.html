TPKC project: Configuration variables

This document details all configuration variables related to the TextPad built-in commands (TPKC) project.

[top]   TPKC project: All categories and category count

Category-list configuration: zTpkcAllCategoryAbrvLCsQbl

zTpkcAllCategoryAbrvLCsQbl defines the universe of command categories in the TextPad application. This set of categories is exactly equal to those in
    TextPad > Configure > Preferences > Keyboard > Category
and in the same order, with the exceptions that it

  • uses abbreviated and lowercase names, and
  • does not include the "All" category.

As of version 7.0.9, its value is
    [file][edit][search][view][tools][macros][config][window][help][cursor][sel][cliplib][chars]

For each value in this sQuare-Bracketed List (QBL), there is a corresponding category configuration.

Note that this all-categories-qbl may be overridden, so that the entire project may be built with an exact subset of these categories, for the purposes of testing and demonstration.

Locations

  • In PhraseExpress:

[top]   TPKC project: Category configuration

Each category is defined by six configuration variables:

  • [go]   zTpkc[cat-name]AbrvLC: Lowercase abbreviation
  • [go]   zTpkc[abrv]AllCmdsQbl: Square Bracketed List of all command names.
  • [go]   zTpkc[abrv]CmdCount: The number of commands in this category.
  • [go]   zTpkc[abrv]HasUserCrtdCmds_yes: Does this category allow user-created commands?
  • [go]   zTpkc[abrv]GTStpsFrmCatLstTp: Navigational steps from the top of the Category-list
  • [go]   zTpkc[abrv]NameLC: The full name of this category, as defined By TextPad
Where '[abrv]' is one of the members of the overall configuration variable
    zTpkcAllCategoryAbrvLCsQbl
and '[cat-name]' is the value of zTpkc[abrv]NameLC

  • Example value: edit
  • See: zTpkc[abrv]NameLC

The LowerCase abbreviation for this category. This is the primary identifier for a category in the TPKC project. It is used for phrase names, file names, and for public-display.

[back]   TPKC category config: zTpkc[abrv]HasUserCrtdCmds_yes

[top]   TPKC project: Command configuration

Every built-in TextPad command has a specific set of "global" and "user-specific" configuration variables.

  • Global command configuration has the same value for all users.
  • User-specific command configuration has potentially-different values for each user, within their specific installation of TextPad.
Global configuration for a command defines everything EXCEPT the key-commands assigned to it. User-configuration defines ONLY the key-commands assigned (or not assigned) to a command.

All global command configuration variables:

  • [go]   zTpkc[name]Abrv1C: Abbreviation used as private-key
  • [go]   zTpkc[abrv]CmdNameTC: Full name as assigned by TextPad
  • [go]   zTpkc[abrv]DfltPhrsNm: The name of the default phrase to trigger this command in TextPad (call its key-command)
  • [go]   zTpkc[abrv]CatForCmdAbrvLC: The category in which this command exists
  • [go]   zTpkc[abrv]CmdType_mcrTl: Of what type is this command?
  • [go]   zTpkc[abrv]IsBtstrp_yes: Is this a bootstrap command?
  • [go]   zTpkc[abrv]DfltPostKcSlpMils: The default number of milliseconds to sleep after the key-command is pressed.
  • [go]   zTpkc[abrv]StpsFrmCmdListTop: The navigational steps to this command from the list-top.

All user-specific command configuration variables:

  • [go]   zzTpkc[name]NbsKcCount: The number of non-bootstrap key-commands.
  • [go]   zzTpkc[name]NbsKcEsc#: A specific and escaped nbs key-command.

Key:

  • [name] is the value of zTpkc[abrv]CmdNameTC
  • [abrv] is the value of zTpkc[name]Abrv1C

[back]   TPKC global config: zTpkc[abrv]CmdType_mcrTl

  • Example name: zTpkcEditCmdType_mcrTl

The type of this command. Either

  • Built-in: Indicated by a value of nothing (the empty-string, ''). By definition, all commands in the "TextPad built-in commands" project are built-in. This field is for future compatibility with macro and tool (user-created) commands.
  • 'macro': A user-created macro
  • 'tool': A user-created tool

Note that the TPKC-Project does not recognize any user-created (or example) commands, such as macros or tools. User-created commands must be created and configured (and their key-commands assigned) manually.

TPKC global config: zTpkc[command-name]Abrv1C     [top]

  • Example phrase name: zzTpkcEditAlignRightAbrv1C
  • Example value: Ealr

The command's 1-case abbreviation, used as the private key (the unique portion of phrase-names) within the TPKC project.

There is a single public-use for this abbreviation: When a single command is to be processed (all of its key-commands assigned, for example) via the
    TextPad built-in commands (TPKC) > 1 user-prompted command branch in the TPKC drop-down menu. The command's abbreviation is prompted for. Note that command-abbreviations can be discovered via the description in every
    TextPad built-in commands (TPKC) > Assign 1 command > [category-name] branch-item.

TPKC global config: zTpkc[abrv]CmdNameTC     [top]

  • Example phrase name: zzTpkcEalrCmdNameTC
  • Example value: EditAlignRight

The name exactly as in
    TextPad > Configure > Preferences > Keyboard > [category-name] > Commands The only exception is the 'characters' category. TextPad does not assign any names to the commands in this category. The character itself IS the command name. Since this is unacceptable within the TPKC project, when creating the raw-global-command-configuration (from the TextPad command category lists), each of these character-commands are assigned a specific name (such as Char16AUpperDegree and Char17ALowerDegree).

TPKC global config: zTpkc[abrv]DfltPhrsNm     [back]

  • Example phrase name: zzTpkcEalrDfltPhrsNm
  • Example value: zTPAlignTextRight

The phrase name that triggers this command (via its 'primary' key-command), and is intended to be called by other PhraseExpress phrases. If there are no key-commands for a command, its default-phrase is never created. If there IS a key-command (either bootstrap, or non-bootstrap), then this phrase is created as final step in the TPKC build process.

TPKC global config: zTpkc[abrv]CatForCmdAbrvLC     [back]

  • Example phrase name: zzTpkcEalrCatForCmdAbrvLC
  • Example value: edit

This value must be one of the members of zTpkcAllCategoryAbrvLCsQbl:
    {#insert zTpkcAllCategoryAbrvLCsQbl}

TPKC global config: zTpkc[abrv]IsBtstrp_yes     [back]

  • Example phrase name: zzTpkcEalrIsBtstrp_yes

If this is a 'boostrap' command, then this must equal 'yes'. Otherwise, this must be nothing (meaning the empty-string, ''). A bootstrap command is a command that is needed before the TPKC project exists, or is needed by the TPKC build-process itself. See ................

TPKC global config: zTpkc[abrv]DfltPostKcSlpMils     [back]

  • Example phrase name: zzTpkcEalrDfltPostKcSlpMils

The default number of milliseconds to sleep after the command's key-command is pressed. This must be a positive integer that is valid for the \\\{#sleep ...\\\} command (greater than zero). This sleep command is implemented in the nbs-phrase, as output by tpkc-build-process 14.

TPKC global config: zTpkc[abrv]StpsFrmCmdListTop     [back]

  • Example phrase name: zzTpkcEalrStpsFrmCmdListTop
  • Example value: \\\{#DOWN -count 3\\\}\\\{#insert z5ThnTSlp\\\}

The specific navigational steps necessary to get to this command in the command list:
    TextPad > Configure > Preferences > Keyboard > [category-name] > Commands

TPKC user-specific config: zzTpkc[name]NbsKcCount     [back]

  • Example phrase-name: zzTpkcFileCloseNbsKcCount

The number of non-boostrap key-commands assigned to this built-in TextPad command. Must be an integer zero or greater. For every number above zero, there must be a corresponding zzTpkc[name]NbsKcEsc# variable. For example, if
    zzTpkcFileCloseNbsKcCount is 3, then the following must exist:

  • zzTpkcFileCloseNbsKcEsc1
  • zzTpkcFileCloseNbsKcEsc2
  • zzTpkcFileCloseNbsKcEsc3
When '0', then all zzTpkcFileCloseNbsKcEsc# variables are ignored.

TPKC user-specific config: zzTpkc[name]NbsKcEsc#     [back]

  • Example phrase-name: zzTpkcFileCloseNbsKcEsc1
  • Example value: \\\\\\{#insert zCpF12\\\\\\}\\\\\\{#sleep 25\\\\\\}\\\\\\{#insert zCpF9\\\\\\}

The specific key-command that triggers (should trigger) the command. This value must be escaped (the escaped version is printed for diagnostic purposes).

There must be one of these variables for every number that zzTpkc[name]NbsKcCount is above zero.
