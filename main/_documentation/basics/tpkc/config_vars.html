[top]   TPKC: Cmd-cfg-vars: zTpkcAllCategoryAbrvLCsQbl

[Go to locations]

The sQuare-Bracketed List (qbl) that defines the universe of command categories in the TextPad application. This set of categories is exactly equal to those in
    TextPad > Configure > Preferences > Keyboard > Categories
and in the same order, with the exceptions that it

  • uses abbreviated and lowercase names, and
  • does not include the "All" category.

As of version 7.0.9, its value is
    [file][edit][search][view][tools][macros][config][window][help][cursor][sel][cliplib][chars]

For each value in this qbl, there is a corresponding set of category configuration.

This category-universe may be overridden by the master-configuration-variable   zTpkcBuildModeTestOrLive.

[top]   TPKC project: Category configuration

[Go to locations]

Each category is defined by six configuration variables:

  • [GO]   zTpkc[cat-name]AbrvLC: Lowercase abbreviation
  • [GO]   zTpkc[abrv]AllCmdsQbl: Square Bracketed List of all command names.
  • [GO]   zTpkc[abrv]CmdCount: The number of commands in this category.
  • [GO]   zTpkc[abrv]HasUserCrtdCmds_yes: Does this category allow user-created commands?
  • [GO]   zTpkc[abrv]GTStpsFrmCatLstTp: Navigational steps from the top of the Category-list
  • [GO]   zTpkc[abrv]NameLC: The full name of this category, as defined By TextPad
Where "[abrv]" is one of the members of
    zTpkcAllCategoryAbrvLCsQbl
and "[cat-name]" is the value of zTpkc[abrv]NameLC

[back]   TPKC category config: zTpkc[abrv]AbrvLC

  • Example name: zTpkcEditAbrvLC
  • Example value: edit
  • See: zTpkc[abrv]NameLC

The LowerCase abbreviation for this category. This is the primary identifier for a category in the TPKC project. It is used for phrase names, file names, and for public-display.

[back]   TPKC category config: zTpkc[abrv]AllCmdsQbl

  • Example name: zTpkcEditAllCmdsQbl
  • Example value: [EditAlignCenter][EditAlignJustify][EditAlignLeft][EditAlignRight][EditBackspace][EditCapitalize][EditClearAll][EditCopy][EditCopyAllFileNames][EditCopyAppend][EditCopyAsHTML][EditCopyFileName][EditCopyLine][EditCopyLineAppend][EditCopyTaggedLines][EditCopyWord][EditCopyWordAppend][EditCut][EditCutAppend][EditCutLine][EditCutLineAppend][EditCutTaggedLines][EditCutWord][EditCutWordAppend][EditDelete][EditDelLine][EditDelLineEnd][EditDelSelection][EditDelTaggedLines][EditDelWord][EditDelWordLeft][EditDelWordRight][EditDupLine][EditFillBlock][EditIMEreconvert][EditIndentLess][EditIndentMore][EditInsertFile][EditInsertFileName][EditInsertStatistics][EditInvertCase][EditJoinLines][EditLongDate][EditLowerCase][EditMarkClean][EditNewLineAfter][EditNewLineBefore][EditPageBreak][EditPaste][EditPasteBlock][EditPasteHTML][EditPasteLines][EditPopupContext][EditPopupDateTime][EditRedo][EditRedoAll][EditReformat][EditSentenceCase][EditShortDate][EditSplitLines][EditTime][EditTranspose][EditTransposeWords][EditUndo][EditUndoAll][EditUpperCase][EmptyClipboard]

The full list of commands in this category (their full names), each surrounded by square brackets (with no spaces between). It is recommended that this be in alphabetical order.

The number of elements in this qb-list must match the value of its corresponding zTpkc[abrv]CmdCount.

[back]   TPKC category config: zTpkc[abrv]CmdCount

  • Example name: zTpkcEditCmdCount
  • Example value: 67

The number of commands in this category's qb-list.

[back]   TPKC category config: zTpkc[abrv]HasUserCrtdCmds_yes

  • Example name: zTpkcEditHasUserCrtdCmds_yes

If the category allows the user to add commands to it (such as a macro or a tool), then set this to "yes". Otherwise, set this to the empty-string ('').

Note that the TPKC project does not recognize any user-created (or example) commands, such as macros or tools. User-created commands must be created and configured (and their key-commands assigned) manually.

[back]   TPKC category config: zTpkc[abrv]NameLC

  • Example name: zTpkcEditNameLC
  • Example value: edit

The full name of this category, exactly as defined in

[top]   TPKC project: Command configuration

[locations]

Every built-in TextPad command has a specific set of "global" and "user-specific" configuration variables.

  • Global command configuration has the same value for all users.
  • User-specific command configuration has potentially-different values for each user, within their specific installation of TextPad.
Global configuration for a command defines everything EXCEPT the key-commands assigned to it. User-configuration defines ONLY the key-commands assigned (or not assigned) to a command.

All global command configuration variables:

  • [GO]   zTpkc[name]Abrv1C: Abbreviation used as private-key
  • [GO]   zTpkc[abrv]CmdNameTC: Full name as assigned by TextPad
  • [GO]   zTpkc[abrv]DfltPhrsNm: The name of the default phrase to trigger this command in TextPad (call its key-command)
  • [GO]   zTpkc[abrv]CatForCmdAbrvLC: The category in which this command exists
  • [GO]   zTpkc[abrv]CmdType_mcrTl: Of what type is this command?
  • [GO]   zTpkc[abrv]IsBtstrp_yes: Is this a bootstrap command?
  • [GO]   zTpkc[abrv]DfltPostKcSlpMils: The default number of milliseconds to sleep after the key-command is pressed.
  • [GO]   zTpkc[abrv]StpsFrmCmdListTop: The navigational steps to this command from the list-top.

Changing any value in the TPKC global-command configuration requires the entire TPKC project to be re-built, from sub-build-process 01.

Key:

  • [name] is the value of zTpkc[abrv]CmdNameTC
  • [abrv] is the value of zTpkc[name]Abrv1C

[back]   TPKC global config: zTpkc[abrv]CmdType_mcrTl

  • Example name: zTpkcEditCmdType_mcrTl

The type of this command. Either

  • Built-in: Indicated by a value of nothing (the empty-string, ''). By definition, all commands in the "TextPad built-in commands" project are built-in. This field is for future compatibility with macro and tool (user-created) commands.
  • "macro": A user-created macro
  • "tool": A user-created tool

Note that the TPKC-Project does not recognize any user-created (or example) commands, such as macros or tools. User-created commands must be created and configured (and their key-commands assigned) manually.

TPKC global config: zTpkc[command-name]Abrv1C     [top]

  • Example phrase name: zzTpkcEditAlignRightAbrv1C
  • Example value: Ealr

The command's 1-case abbreviation, used as the private key (the unique portion of phrase-names) within the TPKC project.

There is a single public-use for this abbreviation: When a single command is to be processed (all of its key-commands assigned, for example) via the
    TextPad built-in commands (TPKC) > 1 user-prompted command
branch in the TPKC drop-down menu. The command's abbreviation is prompted for. Note that command-abbreviations can be discovered via the description in every
    TextPad built-in commands (TPKC) > Assign 1 command > [category-name] branch-item.

TPKC global config: zTpkc[abrv]CmdNameTC     [top]

  • Example phrase name: zzTpkcEalrCmdNameTC
  • Example value: EditAlignRight

The name exactly as in
    TextPad > Configure > Preferences > Keyboard > Categories > [category-name] > Commands
The only exception is the "Characters" category. TextPad does not assign any names to the commands in this category. The character itself IS the command name. Since this is unacceptable within the TPKC project, when creating the raw-global-command-configuration (from the TextPad command category lists), each of these character-commands are assigned a specific name (such as Char16AUpperDegree and Char17ALowerDegree).

TPKC global config: zTpkc[abrv]DfltPhrsNm     [back]

  • Example phrase name: zzTpkcEalrDfltPhrsNm
  • Example value: zTPAlignTextRight

The phrase name that triggers this command (via its primary key-command), and is intended to be called by other PhraseExpress phrases. If there are no key-commands for a command, its default-phrase is never created. If there IS a key-command (either bootstrap, or non-bootstrap), then this phrase is created by the tpkc.second of two TPKC user-build-processes.

TPKC global config: zTpkc[abrv]CatForCmdAbrvLC     [back]

  • Example phrase name: zzTpkcEalrCatForCmdAbrvLC
  • Example value: edit

This value must be one of the members of zTpkcAllCategoryAbrvLCsQbl

TPKC global config: zTpkc[abrv]IsBtstrp_yes     [back]

  • Example phrase name: zzTpkcEalrIsBtstrp_yes

If this is a TextPad-bootstrap, then this must equal 'yes'. Otherwise, this must be nothing (meaning the empty-string, '').

[BACK]   TPKC global config: zTpkc[abrv]DfltPostKcSlpMils

  • Example phrase name: zzTpkcEalrDfltPostKcSlpMils
  • See: The TPKC master-configuration variable
        zTpkcMillsBtwMultiKeyKCs

The default number of milliseconds to sleep after the command's key-command is pressed. This must be a positive integer that is valid for the
    {#sleep ...}
command (greater than zero). This sleep command is implemented in the nbs-phrase, as output by tpkc sub-build-proc-14.

  • Example phrase name: zzTpkcEalrStpsFrmCmdListTop

The specific navigational steps necessary to get to this command in the command list:
