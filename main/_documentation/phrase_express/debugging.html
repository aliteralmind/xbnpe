[top]   XBNPE debugging: Process tracing and freezing with alertbox[1-10]

alertbox-es provide the ability to narrow down the location at which a problem is occuring ("tracing"), and stopping execution temporarily to analyze the current state ("freezing"). Insert alertbox-es them as necessary, using the 'pab[1-10]' actions. For example,
    pab3
is the phrase-creator for the command
    {#insert alertbox3}
which pops up a simple dialog box at that point, containing only the number three.

(Note that alertbox-es are an exception to the 'z'-prefix naming-convention for commands.)

Find all alertbox phrase-calls: fnpab

fnpab is an action that sets the search term to
    alertbox[0-9]+
allowing you to quickly find (and delete) all alertbox-es in the code.

Warning: alertbox-es in the find-dialog changes focus

Note that, when in the find-dialog (or replace or find-in-files dialogs), using an alertbox causes focus to change to the find-what textbox, and for all its text to become selected.

[top]   XBNPE debugging: Changing execution speed (and "zTSlp")

[go to temporary, permanent]

The most powerful diagnostic tool in XBN-PhraseExpress is the ability to slow or speed up execution of anything and everything, whether the code is created by you, or is among the already-existing XBNPE projects and processes. Key-press speed is made possible by the XBNPE-Key-Press (XKP) project. Each x-key-press is equivalent to its default PhraseExpress key-press, with an additional call to
    zTSlp
For example,
    {#insert zCpHOME}
is equal to
    {#CTRL {#HOME}}{#insert zTSlp}

Speed Contents of zTSlp Action Phrase-creator Command
Fast not-applicable kpf kfp zKeyPressFast
During "fast" execution speed, the zTSlp phrase does not exist, and therefore has minimal overhead.
Medium {#sleep 250} kpm kmp zKeyPressMedium
After each key-press, there is a quarter-second pause.
Slow {#sleep 500} kps ksp zKeyPressSlow
Super-Slow {#sleep 1000} kpss kssp zKeyPressSuperSlow
Glacial {#sleep 2000} kpg kgp zKeyPressGlacial

Notes:

  • Each "action" and "command" in the above table immediately changes key-press speed globally--for everything, both in, and created with, XBN-PhraseExpress.
  • Each command is accessible in the zCASpDNR_SLASH user-menu (under the heading "Key-press [execution] speeds"). To determine the current speed, use 'kpt'. kpt also presents a complete syntax to speed-related actions (aside from the below 'fnkp').
  • Aside from fast-mode, each of the above commands are equivalent to
        {#SETPHRASE -description zvMILLS -content [the-sleep-number-as-in-the-above-table] -autotext zvMILLS -folder ctvv_folder}{#insert zKeyPressSpeedSetToMills_M_mils}

[BACK]   To temporarily slow down a portion of your code

(*At the very beginning:*){#insert zKeyPressFast}
   ...stuff that already works...
{#insert zKeyPressSlow}
   ...stuff that needs to be analyzed...
{#insert zKeyPressFast}

To quickly find (and delete) all calls to
    'zKeyPress[anything]'
use 'fnkp' (this is not, and cannot be, mentioned in syntax listing).

[BACK]   To permanently (or to offer your users the possibility to) slow down execution...

...of a portion of your code, yet still respect the currently-chosen global-speed-setting:

   ...stuff that should definitely go fast (by default)...
{#insert zKeyPressSpeedSave}
{#insert zKeyPressSlow}
   ...portion to slow down...
{#insert zKeyPressSpeedRevert}

or:

Allow your user to call any of the following:

  • zKPSpeedRequestFast
  • zKPSpeedRequestMedium
  • zKPSpeedRequestSlow
  • zKPSpeedRequestSuperSlow
  • zKPSpeedRequestGlacial

And then, eventually a process is executed:

{#insert zKPSpeedSaveSetToRqstd}

   ...Do some stuff at the requested speed...

{#insert zKeyPressSpeedRevert}

Note that zKPSpeedRequest* is a global variable. Changing it in one application automatically changes it in another.
