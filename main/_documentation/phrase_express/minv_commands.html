XBN-PhraseExpress: MINV-Commands

A MINV-Command is a phrase that can be called repeatedly--a loop. More than a loop, however, each MINV-Command includes a large set of utilities for executing the command, printing literal phrase-calls to it, and for diagnostics. At the heart of a MINV-Command is its "Master Phrase", and the "MINV-Variable" required by it.

The MINV-Command project is based on the Large Project Database (LPD) Framework.

A MINV-Command provides for three main phrase-types:

  • Commands: Phrases that are triggered by other phrases (such as 'zENTER2' or 'zENTER3', which prints two or three new-lines at the cursor).
  • Actions: Shortcuts to commands, intended for direct use by human beings ('etr5', which prints five new-lines right now).
  • Phrase Creators: Actions that do nothing more than print literal phrase-calls to commands. ('etr{#insert zzMnvSynPstMainPhrsCrtr}2', or 'etr{#insert zzMnvSynPstMainPhrsCrtr}3', the latter of which results in
        \\\{#insert zENTER3\\\})
Each of these three phrase-types are optional, but at least one is required.

Introduction by example: 'etr#'     [top]

This section will demonstrate a MINV-Command that simply prints one-or-more newlines at the cursor. It will be abbreviated 'etr'. Unlike 'hobk' in the video, this etr command is not an action--why type 'e' 't' 'r' [trigger-char], when you could just press the enter key?

The 'etr#' command defines a range of commands that can be called from other phrases. We'll define the most common numbers to be one-through-five--this is its "Command Range". This means that all the following phrases are avaliable:
    -zENTER1
    \\\{#insert zvLINES_DOWN_st1\\\}\\\{#insert zENTERN_M_vlnd\\\}
    -zENTER2
    \\\{#insert zvLINES_DOWN_st2\\\}\\\{#insert zENTERN_M_vlnd\\\}
    -zENTER3
    \\\{#insert zvLINES_DOWN_st3\\\}\\\{#insert zENTERN_M_vlnd\\\}
    -zENTER4
    \\\{#insert zvLINES_DOWN_st4\\\}\\\{#insert zENTERN_M_vlnd\\\}
    -zENTER5
    \\\{#insert zvLINES_DOWN_st5\\\}\\\{#insert zENTERN_M_vlnd\\\}

Where zvLINES_DOWN_st2 sets the variable 'zvLINES_DOWN' to 2
    \\\{#SETPHRASE -description zvLINES_DOWN -content 1 -autotext zvLINES_DOWN -folder ctvv_folder\\\}
and zENTERN_M_vlnd is equal to
    \\\{#LOOP \\\{#ENTER\\\} -count \\\{#insert zvLINES_DOWN\\\}}

So, ultimately, this MINV-Command is a simple loop, where enter is pressed multiple times, with the number of presses defined by zvLINES_DOWN.

Realistically, these commands would be implemented as a direct calls, such as
    -zENTER1
    \\\{#ENTER\\\}
    -zENTER2
    \\\{#ENTER -count 2\\\}

Introduction: Master Phrase

All MINV-Commands have a Master Phrase, which, generally speaking, is the single location where all logic resides. The master-phrase for 'etr#' is
    -zENTERN_M_vlnd
    \\\{#LOOP \\\{#ENTER\\\} -count \\\{#insert zvLINES_DOWN\\\}}

(Note that
    \\\{#ENTER -count \\\{#insert zvLINES_DOWN\\\}}
does not work.)

The end of the master-phrase's name is "_M_vlnd", which is an indication that the MINV-Variable needed by this command is named
    zvLINES_DOWN
('vlnd' is its abbreviation, as configured elsewhere). This is just a naming convention I've decided on, for both in and out of the MINV-project.

Every master-phrase requires a single MINV-Variable in order to function properly.

The MINV master-phrase allows for a MINV-Command to be executed with any number (1 or greater), without having to write additional phrases. The most-commonly-needed numbers are defined by the command-range (1-5) and, for each of these numbers, a specific command (zENTER[1-5]) and phrase-creator (etr{#insert zzMnvSynPstMainPhrsCrtr}[1-5]) is provided.

(Were this command also an "action", then etr[1-5] would also be avaliable--conversely, actions are the only kind of phrases available in 'hobk', as demonstrated in the above video.)

Unusual numbers can also be executed, by setting the MINV-Variable
    \\\{#SETPHRASE -description zvLINES_DOWN -content 15 -autotext zvLINES_DOWN -folder ctvv_folder\\\}
and then calling the master-phrase.
    \\\{#insert zENTERN_M_vlnd\\\}

There are numerous utilities to help you create these phrase-calls. See the next section, as well as the "phrase-creators" listed in the "Syntax summary: Overview" section.

In addition, the master-phrase also has a literal version of itself:
    -zENTERN_M_vlndp
    \\\{#insert zENTERN_M_vlnd\\\}
as well as doubly-literal:
    -zENTERN_M_vlndq
    \\\{#insert zENTERN_M_vlndp\\\}
which are used in phrase-creators, as described in the intro-putting-it-together section.

Introduction: Phrase Creators

A "Phrase Creator" assists you in writing PhraseExpress code. A MINV-Command can provide both regular and literal phrase-creators.

A regular phrase-creator prints out a literal phrase-call to a MINV-Command. For example, 'etr{#insert zzMnvSynPstMainPhrsCrtr}2' prints this at the cursor:
    \\\{#insert zENTER2\\\}
A literal phrase-creator prints out a doubly-literal phrase-call: 'etr{#insert zzMnvSynPstLtrlPhrsCrtr}3' prints
    \\\\\\{#insert zENTER3\\\\\\}

Phrase-creators use the same command-range (which, for 'etr#', is 1-5), so here are available regular phrase-creators:

  • 'etr{#insert zzMnvSynPstMainPhrsCrtr}1' which prints out \\\{#insert zENTER1\\\}
  • 'etr{#insert zzMnvSynPstMainPhrsCrtr}2': \\\{#insert zENTER2\\\}
  • 'etr{#insert zzMnvSynPstMainPhrsCrtr}3': \\\{#insert zENTER3\\\}
  • 'etr{#insert zzMnvSynPstMainPhrsCrtr}4': \\\{#insert zENTER4\\\}
  • 'etr{#insert zzMnvSynPstMainPhrsCrtr}5': \\\{#insert zENTER5\\\}
and all literal phrase-creators:
  • 'etr{#insert zzMnvSynPstLtrlPhrsCrtr}1' which prints \\\\\\{#insert zENTER1\\\\\\}
  • 'etr{#insert zzMnvSynPstLtrlPhrsCrtr}2': \\\\\\{#insert zENTER2\\\\\\}
  • 'etr{#insert zzMnvSynPstLtrlPhrsCrtr}3': \\\\\\{#insert zENTER3\\\\\\}
  • 'etr{#insert zzMnvSynPstLtrlPhrsCrtr}4': \\\\\\{#insert zENTER4\\\\\\}
  • 'etr{#insert zzMnvSynPstLtrlPhrsCrtr}5': \\\\\\{#insert zENTER5\\\\\\}

For more information about phrase-creators, see the '{#insert zzMnvSynPstNumSetphrsCallMstr}', '{#insert zzMnvSynPstSetToNumCallMstr}', and '{#insert zzMnvSynPstSetphrsToPromptCallMstr}' commands, as well as the [PhraseCreators_on] configuration-variable.

Introduction: MINV-Variable

Each MINV-Command uses a single "MINV-Variable". A MINV-Variable is nothing more than a phrase where the entire body is an integer (1 or greater). Specifically, the master-phrase of a MINV-Command requires a particular MINV-Variable to run properly.
    zENTERN_M_vlnd
requires
    zvLINES_DOWN
as denoted by the "_M_vlnd" ('vlnd' is its configured abbreviation).

A MINV-Variable, however, may be used by multiple MINV-Commands. While each command has its own command-range, each MINV-Variable has a "Variable Range". The variable-range is a superset of all the command ranges, of all those MINV-Commands that use this variable. In other words, the command-range of every command that uses this variable, must be fully, and inclusively, contained in the variable-range.

The variable range of zvLINES_DOWN is configured to be 1 through 10. There is an action and phrase-creator utility for each number in the range, which is a shortcut to the appropriate SETPHRASE. Here are all available actions:

  • -zvLINES_DOWN_st1
    \\\{#SETPHRASE -description zvLINES_DOWN -content 1 -autotext zvLINES_DOWN -folder ctvv_folder\\\}
  • -zvLINES_DOWN_st2
    \\\{#SETPHRASE -description zvLINES_DOWN -content 2 -autotext zvLINES_DOWN -folder ctvv_folder\\\}
  • And so on: zvLINES_DOWN_st3, zvLINES_DOWN_st4, zvLINES_DOWN_st5, zvLINES_DOWN_st6, zvLINES_DOWN_st7, zvLINES_DOWN_st8, zvLINES_DOWN_st9, zvLINES_DOWN_st10
phrase-creators:
  • zvLINES_DOWN_st1p is equal to
        \\\\\\{#SETPHRASE -description zvLINES_DOWN -content 1 -autotext zvLINES_DOWN -folder ctvv_folder\\\\\\}
  • And so on: zvLINES_DOWN_st2p, zvLINES_DOWN_st3p, zvLINES_DOWN_st4p, zvLINES_DOWN_st5p, zvLINES_DOWN_st6p, zvLINES_DOWN_st7p, zvLINES_DOWN_st8p, zvLINES_DOWN_st9p, zvLINES_DOWN_st10p
and literal phrase-creators:
  • zvLINES_DOWN_st1q is equal to
        \\\\\\{#SETPHRASE -description zvLINES_DOWN -content 1 -autotext zvLINES_DOWN -folder ctvv_folder\\\\\\}
  • And so on: zvLINES_DOWN_st2q, zvLINES_DOWN_st3q, zvLINES_DOWN_st4q, zvLINES_DOWN_st5q, zvLINES_DOWN_st6q, zvLINES_DOWN_st7q, zvLINES_DOWN_st8q, zvLINES_DOWN_st9q, zvLINES_DOWN_st10q

Introduction: Putting it together     [top]

To circle back around to MINV-Commands: All MINV-Commands take advantage of these '_st#' phrases:

  • Command: 'zENTER2' is generally expected to equal
        \\\{#insert zvLINES_DOWN_st2\\\}\\\{#insert zENTERN_M_vlnd\\\}
  • And, when there are no commands:
    • Action: 'etr2':
          \\\{#insert zvLINES_DOWN_st2\\\}\\\{#insert zENTERN_M_vlnd\\\}
    • Phrase-creator: 'etr{#insert zzMnvSynPstMainPhrsCrtr}2':
          \\\{#insert zvLINES_DOWN_st2p\\\}\\\{#insert zENTERN_M_vlndp\\\}
    • Literal phrase-creator: 'etr{#insert zzMnvSynPstLtrlPhrsCrtr}2':
          \\\{#insert zvLINES_DOWN_st2q\\\}\\\{#insert zENTERN_M_vlndq\\\}
(These are the phrase's contents--their uninterpreted phrase-bodies.)

Syntax sumary: Overview

The master-phrase description contains a syntax summary of all the available MINV-Commands:

zENTERN_M_vlnd -(**)- etr[p,q]#, etr[f,[[h,t,r]]m], vlnd[[p,f]]

Specifically, this syntax implies the following:

  • etr[p,q]#   [go]
    • etr{#insert zzMnvSynPstMainPhrsCrtr}#: Phrase-creator
    • etr{#insert zzMnvSynPstLtrlPhrsCrtr}#: Literal phrase-creator
  • etr[f,[[h,t,r]]m]   [go]
    • etr{#insert zzMnvSynPstCmdInfoSyntax}: inFormation
    • Phrase-creators:
      • etr{#insert zzMnvSynPstNumSetphrsCallMstr}
      • etr{#insert zzMnvSynPstSetToNumCallMstr}
      • etr{#insert zzMnvSynPstSetphrsToPromptCallMstr}
  • vlnd[[p,f]]: MINV-Variable utilities   [go]
    • vlnd: Prints the current value (for diagonstics)
    • vlndp: Phrase-creator
    • vlndf: inFormation

For in-depth documentation on how this syntax-summary--and the 'etr[p,q]#' portion in particular--is generated, see "Technical documentation: Syntax for a MINV-Command Master-Phrase's description"

Detail: etr[p,q]#     [back]

  • etr{#insert zzMnvSynPstMainPhrsCrtr}#: Phrase-creator
  • etr{#insert zzMnvSynPstLtrlPhrsCrtr}#: Literal phrase-creator

Depending on how you configure you MINV-Command, this is the one portion of the syntax-summary that may be different. For example, were this MINV-Command also an action, this syntax would instead be
    etr[[{#insert zzMnvSynPstMainPhrsCrtr},{#insert zzMnvSynPstLtrlPhrsCrtr}]]#
meaning the '{#insert zzMnvSynPstMainPhrsCrtr}'-or-'{#insert zzMnvSynPstLtrlPhrsCrtr}' is optional--and therefore the action
    etr#
(such as 'etr3') is also available.

etr{#insert zzMnvSynPstMainPhrsCrtr}#

MINV-Command cv-type: [ActionName1C] [back]

  • Example value: 'Etr'
  • Requirements: Must be alpha-numeric, at least one character in length, where the first character is uppercase, and all others are lowercase. Must be unique among all MINV-Commands.
  • Recommendation: Should be as short-as-possible, as long as it is easy to remember.
  • Internal variable name: zvACTION_NM_1C

This MINV-Command's Public Key. This is the unique identifier for a MINV-Command, as used by you and your users. Whenever a process is run for, or on, a specific MINV-Command, this public key is always the first piece of information needed.

In most cases, only the lowercase version is needed. However, it is stored internally with the first character ['1C'] as uppercase.

The term "key" here refers to a database--where each data-instance, or row in a table, has a uniquely-identifying primary key. This "public" and "private" key terminology is not related in any way to encryption.

MINV-Command cv-type: [MasterPhraseNm]   [back]

  • Example value: zENTERN_M_vlnd
  • Requirements: Must be alpha-numeric, and at least one character in length. Must be unique among all MINV-Commands.
  • Internal variable name: zvMSTR_PHRS_NM

The full name (and autotext) of the MINV-Command's Master Phrase. The master-phrase is, generally speaking, the single location where all logic for a MINV-Command is written.

It is recommended that this name be equal to
    z[NamePreNum]N[NamePostNum]_M_[VarAbrv1CForCmd]
When commands are off, and therefore there is no pre- or post-num name, then its name should be short-but-descriptive. It is always recommended to have the 'N' and '_M_[VarAbrv1CForCmd]'.

'_M_[VarAbrv1CForCmd]' is an XBN-PhraseExpress naming convention, that implies that the phrase requires the variable, whose abbreviation is '[VarAbrv1CForCmd]', to be properly set, in order for this function to work as documented. It is strongly recommended that this variable-abbreviation should be equal to the value of this MINV-Command's [VarAbrv1CForCmd] (except lowercase).

MINV-Command cv-type: [RangeMin] and [RangeMax]   [back]

  • Example values: min=1, max=5
  • Requirements: Both must be an integer greater-than-or-equal-to one. [RangeMax] must be greater-than-or-equal-to [RangeMin], and both must be fully-contained, inclusively, in the MINV-Variable's range.
  • Internal variable names: zvCMD_RANGE_MIN and zvCMD_RANGE_MAX

As described in the introduction, this is the MINV-Command's range of most-commonly-used numbers. For each number in this range there is provided a shortcut command, action, and phrase-creator. Such as, when the range is min=1, max=5, then the available commands (assuming commands are on) are, for example:
    zENTER1, zENTER2, zENTER3, zENTER4, zENTER5
actions (when on):
    etr1, etr2, etr3, etr4, etr5
and phrase-creators (when on):
    etr{#insert zzMnvSynPstMainPhrsCrtr}1, etr{#insert zzMnvSynPstMainPhrsCrtr}2, etr{#insert zzMnvSynPstMainPhrsCrtr}3, etr{#insert zzMnvSynPstMainPhrsCrtr}4, etr{#insert zzMnvSynPstMainPhrsCrtr}5

The smallest-possible range for a MINV-Command is where the minimum and maximum are equal (all likely '1').

MINV-Command cv-type: [VarAbrv1CForCmd]   [back]

  • Example value: Vlnd
  • Requirements: Must be equal to the value of the needed MINV-Variable's (public-key) abbreviation.
  • Internal variable name: zvVAR_ABRV_1C

The public key of the MINV-Variable required by this MINV-Command. This is the link between a command and it's variable. Where each MINV-Command uses exactly one MINV-Variable, each variable may be used by multiple commands. It is strongly recommended that this value (in lowercase form) be at the end of the master-phrase name (for example 'zENTERN_M_vlnd').

Specifically, this is equal to the value of a MINV-Variable's public-key, as defined by its zzMnvVar[private-key]Abrv1C config-var. For example, the MINV-Variable whose public key is named zzMnvVarVlndAbrv1C which has a value--phrase-body--of 'Vlnd' (in this case, the private-key is equal to the public-key).

[Commands_on] sub-cv-type: [NNLCmdXtrnl_yes]   [back]

  • See: [NNLCmdIfIntrnl_yes]

If the no-number-alias of this MINV-Command is created somewhere other than the file in which the master-phrase and commands exist, then it is considered external. An example is 'zENTER', which is a fundamental phrase (essentially equal to \\\{#ENTER\\\}), that existed before the MINV-Command project was ever conceived. This is its external no-number-alias.

When external, set this to 'yes' (no quotes). When internal, set this to nothing (empty-string, '', meaning no.)

A 'yes' value causes the lowest-number command (the command-shortcut having the number equal to [RangeMin], such as 'zENTER1') to be an alias to the external no-num-alias. For example:

    \\\{#insert zENTER\\\}
in this case, this lowest-numbered command will not be one of the command descriptions printed via 'mnvcmp'. Instead, it will be printed as part of the 'mnvcag' (auto-generated) process.

    -zENTER1 -- etr[p]1

MINV-Command cv-type: [Actions_on]   [back]

Sub cv-type (only used when 'Actions_on' is 'on'):

  • SNTPromptActn_on   [go] -- Sub cv-type (only used when [SNTPromptActn_on] is 'on'):
    • SNTPromptActnMsg   [go]
  • Internal variable name: zvACTIONS_ON

Should action-phrases be printed? If yes, then set this to 'on'. Otherwise, set this to the empty-string (nothing, '', meaning off).

An "action" is something that is done right now. It is not something that is called from another phrase. Rather, it is intended to be called by you--a human*. For example, the 'etr#' MINV-Command from the introduction is not an action--hence, zzMnvCmdEtrActions_on is off. (Why type 'e' 't' 'r' [trigger-char], when you could just press the ENTER button?)

However, were this configuration-variable turned 'on' (and assuming a command range of one-to-five), the following actions would be created:
    etr1, etr2, etr3, etr4, etr5
Each of these is equal to the appropriate zENTER# command ('etr4' is equivalent to 'zENTER4'). The action-name is defined by [ActionName1C].

Action-only example: The introduction video discusses a MINV-Command ('hobk', for repeated home-backspaces, for joining multiple lines) that is only an action--something that is only useful right now. It is not useful to any other phrases.

*(While an action-phrase can obviously be called from another phrase, the intention is that the name of an action-phrase is a shortcut--an abbreviation intended for temporary-use. Command-names, on the other hand, are intended to be more explicit and descriptive, hence making the code that might use them more self-documenting.)

This prompting occurs in a simple \\\{#INPUT...\\\} (single-line) dialog.

MINV-Command cv-type: [PhraseCreators_on]   [back]

Sub cv-types (applicable only when [PhraseCreators_on] are 'on'):

  • [PCActOffPst_pES]   [go]
  • [LiteralStyle_esyc]   [go]
  • Internal variable name: zvPHRS_CRTRS_ON

If phrase-creators should be printed, set this to 'on'. Otherwise, set this to nothing (the empty-string, '', meaning off).

As stated in the introduction, a Phrase Creator is a utility for quickly generating PhraseExpress code. Specifically, it helps you to easily create calls to this MINV-Command, for any number--regardless if that number is in or out of the command range.

For the 'etr#' MINV-Command, when [PhraseCreators_on] is 'on' (and assuming a command range of one-to-five), the following phrase-creators are available:
    etr{#insert zzMnvSynPstMainPhrsCrtr}1, etr{#insert zzMnvSynPstMainPhrsCrtr}2, etr{#insert zzMnvSynPstMainPhrsCrtr}3, etr{#insert zzMnvSynPstMainPhrsCrtr}4, etr{#insert zzMnvSynPstMainPhrsCrtr}5
Each of these prints out a literal phrase-call to the appropriate zENTER# command. For example, 'etr{#insert zzMnvSynPstMainPhrsCrtr}4' prints this at the cursor
    \\\{#insert zENTER4\\\}
The action-name--the 'etr' portion of 'etr{#insert zzMnvSynPstMainPhrsCrtr}4'--is defined by [ActionName1C].

For printing phrase-calls outside of the command range, use the '{#insert zzMnvSynPstNumSetphrsCallMstr}', '{#insert zzMnvSynPstSetToNumCallMstr}', and 'rm' commands.

Note that the resulting phrase-calls for the default phrase-creators--for the numbers in the command range--such as 'etr{#insert zzMnvSynPstMainPhrsCrtr}4'...
    \\\{#insert zENTER4\\\}
...are much more concise when compared to the output of the '{#insert zzMnvSynPstNumSetphrsCallMstr}' command:
    \\\{#SETPHRASE -description zvLINES_DOWN -content 4 -autotext zvLINES_DOWN -folder ctvv_folder\\\}\\\{#insert zENTERN_M_vlnd\\\}
Note that this is especially true when commands are 'on', as they are in this example.

Even when commands are off, phrase creators are still more concise. For example, when commands are off, here is the output for both 'etr{#insert zzMnvSynPstMainPhrsCrtr}4' and for 'etr{#insert zzMnvSynPstSetToNumCallMstr}' (when answering '4' at the prompt):
    \\\{#insert zvLINES_DOWN_st4\\\}\\\{#insert zENTERN_M_vlnd\\\}

  • Only applicable when: [PhraseCreators_on] is 'on' and [Actions_on] is off (equal to the empty-string--'')
  • Internal variable name: zvPCAF_PST_PORES

If the name of basic phrase-creators should be postfixed with '{#insert zzMnvSynPstMainPhrsCrtr}', then set this to '{#insert zzMnvSynPstMainPhrsCrtr}' (without the quotes). Otherwise, set this to the empty-string ('', or nothing).

Setting this to the empty-string is a nice shortcut for the most commonly used non-action MINV-Commands (which I actually use for 'etr', 'bsk' [the backspace char], 'dle' [delete], and 'tbb' [tab]).

Examples

When [PCActOffPst_pES] is equal to '{#insert zzMnvSynPstMainPhrsCrtr}', the name of basic phrase-creators is '[ActionName1C]{#insert zzMnvSynPstMainPhrsCrtr}#' (such as 'etr{#insert zzMnvSynPstMainPhrsCrtr}3', which prints this at the cursor:
    \\\{#insert zENTER3\\\}
), and the name of literal phrase creators is '[ActionName1C]{#insert zzMnvSynPstLtrlPhrsCrtr}#' (such as 'etr{#insert zzMnvSynPstLtrlPhrsCrtr}3', which prints either this [when the literal-style is 'curly']:
    \\\\\\{#insert zENTER3\\\\\\}
or [when the literal-style is 'command']:
    \\\{#insert zENTER3{#insert zzMnvSynPstMainPhrsCrtr}\\\}
).

When, however, [PCActOffPst_pES] is equal to the empty-string (''--or nothing), then name of basic phrase-creators is equal to the action-name (such as 'etr3', which prints this at the cursor:
    \\\{#insert zENTER3\\\}
), and the name of literal phrase creators is '[ActionName1C]{#insert zzMnvSynPstMainPhrsCrtr}#' (such as 'etr{#insert zzMnvSynPstMainPhrsCrtr}3', which has the same output as the above 'etr{#insert zzMnvSynPstLtrlPhrsCrtr}3').

[PhraseCreators_on] sub-cv-type: [LiteralStyle_esyc]   [back]

  • Only applicable when: [PhraseCreators_on] is 'on'.
  • Internal variable name: zvLTRL_STYL_ESYC

If there should be only basic phrase-creators (see [PCActOffPst_pES] for examples of both basic and literal phrase creators), then set this to the empty-string ('', or nothing, meaning off). When literal phrase-creators are wanted, then set this to either 'curly' or 'command'.

Curly-style

A 'curly'-style literal phrase-call is one that simply escapes the open- and close-curlys. Such as 'etrq3', which prints this at the cursor:
    \\\\\\{#insert zENTER3\\\\\\}

Command-style

A 'command' style literal-phrase call is recommended only for the most-commonly-used MINV-Commands, such as 'etr', 'bsk' (printing the backspace key multiple times), 'dle' (delete), and 'tbb' (tab). The benefit of a 'command'-style literal phrase-call, is that the six escape characters ('\') are replaced with a single '{#insert zzMnvSynPstMainPhrsCrtr}'. For example, 'etrq3' prints this
    \\\{#insert zENTER3p\\\}
This is made possible by creating a duplicate set of commands, giving each a '{#insert zzMnvSynPstMainPhrsCrtr}'-postfix. For example, if these are the default commands (assuming a command range of 1-5)
    zENTER1, zENTER2, zENTER3, zENTER4, zENTER5
then these extra commands are automatically created (as part of the 'mnvcag' process)
    zENTER1{#insert zzMnvSynPstMainPhrsCrtr}, zENTER2{#insert zzMnvSynPstMainPhrsCrtr}, zENTER3{#insert zzMnvSynPstMainPhrsCrtr}, zENTER4{#insert zzMnvSynPstMainPhrsCrtr}, zENTER5{#insert zzMnvSynPstMainPhrsCrtr}
where each is equal to a phrase-call to it's non-{#insert zzMnvSynPstMainPhrsCrtr} cousin. For example, 'zENTER4{#insert zzMnvSynPstMainPhrsCrtr}' is equal to
    \\\\\\{#insert zENTER4\\\\\\}
and therefore prints out
    \\\{#insert zENTER4\\\}

MINV-Command cv-type: [NoNumAlias_on]   [back]

  • See: [NNLCmdXtrnl_yes], [NNLCmdIfIntrnl_yes]

If a no-number alias is desired, set this to 'on'. Otherwise, set this to the empty-string (nothing--''--meaning off).

There is a potential no-number-alias for each of the three main phrase-types: commands, actions, and phrase-creators. When a particular type is off, then its no-number-alias is not printed (when commands are off, there is no no-number-command).

The main actions (as well as commands and phrase-creators) each have a number, as defined by the command range. For example:
    etr1, etr2, etr3, etr4, etr5
The no-number-alias command is etr, and is equivalent to
    \\\{#insert zENTER[NNLTargetNum]\\\}

[NoNumAlias_on] sub-cv-type: [NNLTargetNum]   [back]

  • Only applicable when: [NoNumAlias_on] is 'on'
  • Internal variable name: zvNNL_TGT_NUM

When no-number-aliases are 'on', then this is the target number for it. The name of the no-number alias has no number, of course, so it needs to explicitely be given a target.

Requirements: This must equal a number in the command range. The only exception is when commands are 'on', in which case this may also equal nothing (the empty-string--''). In the latter situation, the target for both the no-num-action and no-num-phrase-creator is the no-num-command.

When the target-number is empty-string, then only the description of the no-number-command is automatically generated (by 'mnvcmp')--its phrase-body is manually created by you. Therefore the no-num-command is targeted by you (to, in most cases, either the master-phrase or an existing numbered-command).

When the target-number is a number, then the no-num-command is automatically generated by 'mnvcag'.

In all cases, however, the no-num-action and no-num-phrase-creator are both automatically generated by 'mnvcag'.

The universe of configuration-variables that make up a MINV-Variable     [top]

There are exactly five cv-types that make up a MINV-Variable:

  • [go] PNAbrv1C

MINV-Variable cv-type: [PNAbrv1C]   [back]

  • Example value: 'Vlnd'
  • Requirements: Must be alpha-numeric, at least one character in length, where the first character is uppercase, and all others are lowercase. Must be unique among all MINV-Variables.
  • Recommendation: Should be no more than three-or-four characters in length. When this public-key is short, this should be equal to it. When longer, this should be an abbreviated version of it.
  • Internal variable name: zvVAR_PNMV_1C

The MINV-Variable's private-key, which is only used in the phrase-names of this variable's configuration-variables (to distinguish the config-vars of one variable from another). The variable-abbreviation is the public key. Once configured, this is never again used by you, and should never be altered.

MINV-Variable cv-type: [Abrv1C]   [back]

  • Example value: 'Vlnd'
  • Requirements: Must be alpha-numeric, at least one character in length, where the first character is uppercase, and all others are lowercase. Must be unique among all MINV-Variables.
  • Recommendation: Should be as short-as-possible, as long as it is easy to remember.
  • Internal variable name: zvVAR_ABRV_1C

This MINV-Variable's Public Key. This is the unique identifier for a MINV-Variable, as used by you and your users. Whenever a process is run for, or on, a specific MINV-Command, this public key is always the first piece of information needed.

In most cases, only the lowercase version is needed. However, it is stored internally with the first character ['1C'] as uppercase.

MINV-Variable cv-type: [IntrnlVarName]   [back]

  • Example value: 'zvLINES_DOWN'
  • Requirements: Must be at least one character in length, and unique among all MINV-Variables.
  • Recommendation: It is strongly recommended that this begin with 'zv'--a naming-convention meaning temporary variable--and be followed by all-capital letters, divided by underscores. It should be reasonably descriptive, but terse.
  • Internal variable name: zvVAR_IVAR_NAME

The name of the "Temporary Internal Variable" for this MINV-Variable. This is the name of the phrase that will store the number (integer), as needed by the MINV-Command's master-phrase. While this is just a normal phrase like any other, it is considered a "temporary internal variable", because it treaated as temporary, because it is only used during a process (such as a call to 'etr3', during which the temporary variable 'zvLINES_DOWN' is set to '3'). Once the process is complete, this variable may be safely deleted or altered.

MINV-Variable cv-type: [RangeMin] and [RangeMax]   [back]

  • Requirements: Must be an integer greater than zero, and min must be less-than-or-equal-to max. In addition, this range must fully contain (inclusively) every command-range, of every MINV-Command that uses this variable. In other words, the variable's min must be less-than-or-equal-to every command's min, and the variable's max must be greater-than-or-equal-to every command's max.
  • Internal variable names: zvVAR_RANGE_MIN and zvVAR_RANGE_MAX

This is the MINV-Variable's "Variable Range", which defines which set-to-number ("_st#") phrases are printed. For example, a minimum of 1 and maximum of 10 results in these set-to-number phrases being created by 'mnvvag': zvLINES_DOWN_st[1..10]

  • PC=Phrase-Creators 'on' or off? (zvPHRS_CRTRS_ON)
  • p/es=Phrase-creator postfix '{#insert zzMnvSynPstMainPhrsCrtr}' or '' (empty-string)? (zvPCAF_PST_PORES)
  • ltry=Literal phrase-creator style off or on? (zvLTRL_STYL_ESYC) (note that on simply means not off--in other words: 'curly' or 'command')

Notes:

  • sntpa is not applicable when actions (act) are off
  • ltry is not applicable when phrase-creators are off
  • p/es is not applicable unless phrase-creators are 'on' and actions are off

(The asterisk, '*', indicates that the phrase-creator has no postfix--its name is entirely equal to the action-name. Since this is only possible when actions are off, this is not a conflict. This point is reiterated in the master-phrase's body comment.)

These are all the possible commands, some of which are not available, depending on the above syntax (note that a number between [CmdRangeMin] and [CmdRangeMax] can be appended where '#' is printed below--and when [NoNumAlias_onff] is 'on', then no number (the empty-string--''--nothing) is also legal in its place):

  • etr#: Executes the command zENTER# (unless actions are off and the pc-postfix is '', in which case this is instead the phrase-creator)
  • etr{#insert zzMnvSynPstMainPhrsCrtr}#: Prints a call to the command, when phrase-creators are 'on', and '{#insert zzMnvSynPstMainPhrsCrtr}' is the phrase-creator postfix. When commands are off, this is instead equivalent to 'etr{#insert zzMnvSynPstSetToNumCallMstr}'
  • etr{#insert zzMnvSynPstSetNumToPrompt}: Run the set-number-to-prompt action
  • etr{#insert zzMnvSynPstCmdInfoSyntax}: Prints diagnostic information for the MINV-Command
  • etr{#insert zzMnvSynPstNumSetphrsCallMstr}: Prints a literal SETPHRASE for the MINV-Variable (you are prompted for the number), and then a literal-call to the master-phrase
  • etr{#insert zzMnvSynPstSetToNumCallMstr}: Prints a literal-call to zvLINES_DOWN_st# (you are prompted for the number), and then to the master-phrase
  • etr{#insert zzMnvSynPstSetphrsToPromptCallMstr}: Prints a literal-call to zvLINES_DOWN_stPrompt, and then to the master-phrase
  • etr{#insert zzMnvSynPstMstrPhrsNm}: Prints a literal phrase-call to the master-phrase
  • vlnd: Prints the value of the MINV-Variable (for diagnostics)
  • vlnd{#insert zzMnvSynPstMainPhrsCrtr}: Prints a literal phrase-call to the MINV-Variable: \\\{#insert zvLINES_DOWN\\\}
  • vlnd{#insert zzMnvSynPstVarInfoSyntax}: Prints diagnostic information for the MINV-Variable zvLINES_DOWN
