XBN-PhraseExpress: Creating and managing PhraseExpress code

CPCB: Call phrase-name in ClipBoard

'cpcb' is an action phrase-type that executes the phrase-name currently in the clipboard, and also can execute a very limited set of raw PhraseExpress code. Complete 'cpcb' syntax (via 'cpcbsyn') is displayed after executing the cpcb phrase itself.

Contents:

  • [go]   cpcb: The "main" CPCB utility
  • [go]   cpw[l,r] and cpl[n[n],q]: Phrase-name is next to the cursor

While all examples in this documentation assume you are working in a TextPad document, cpcb is useful (and is allowed to be used) in any application, or in Microsoft Windows itself. The phrase name being executed still (?) respects its "Execute only in program" permissions.

[top]   cpcb: The "main" CPCB utility

Executing 'cpcb', in turn, executes the phrase whose name is in the clipboard. For example, with
    zTPFindDlgOpen
in the clipboard, the find dialog opens. It concludes by displaying a BALLOON containing complete cpcb-related syntax.

All other cpcb variations, as listed throughout this document, ultimately call cpcb--or, more accurately, its "raw" command: 'zCallPhraseNameInCB', which does not display the syntax balloon.

[x] option -- Suppressing the syntax-summary balloon

To suppress the syntax-balloon, add the 'x' option: 'cpcbx'. The [x] option is required when the process being called happens to also end with a balloon. Without it, the cpcb-s balloon suppresses it.

Using cpcb to execute code directly

cpcb also works when some very-limited types of raw PhraseExpress code are in the clipboard. Such as
    {#MSGBOX -head Hello -text there}
or
    {#BALLOON -title Please wait... -text Patience!}
or
    {#SETPHRASE -description zvNUMBER -content 5 -autotext zvNUMBER -folder ctvv_folder}

Note that with
    {#SETPHRASE -description zvNUMBER -content 5 -autotext zvNUMBER -folder ctvv_folder}{#insert zvNUMBER}
in the clipboard and executed via cpcb, while zvNUMBER is indeed set to five, it is not output.

[top]   cpw[l,r] and cpl[n[n],q]: Phrase-name is next to the cursor

Copies the word immediately next to the cursor, returns the cursor to its original position, and executes cbcb. Adding the 'x' option instead executes cpcbx

  • cpwl: Copy word-left, cpcb
    I use this in particular, when debugging large processes. I put the processes phrase-name into a new document, place the cursor after it, and execute 'cpwlx' to start it.
  • cpwr: Copy word-right, cpcb
  • cpln: Copy word-left, {#ENTER}, cpcb
  • cplq: Copy word-left, type '=', cpcb

cplnn

Another useful debugging utility. Calls cpln for a "line-list" of phrase-names. Execute this with the cursor immediately-following the top-most phrase-name. For example, executing 'cplnn' with these (bogus! non-existing!) phrase-names... 

zPrintAnX[CURSOR]
zPrintAnX
zPrintTwoXs
zPrintThreeYs

...outputs this:

zPrintAnX
x
zPrintAnX
x
zPrintTwoXs
xx
zPrintThreeYs
yyy[CURSOR]
