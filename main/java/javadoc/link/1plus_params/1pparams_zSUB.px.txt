This comment-block is a design document that is a milestone for me. Above in the left-column are PhraseExpress macros that translate a with-parameters (more-than-zero parameters) function signature, such as
   NumberInRange(NumberBound<N> nb_min, NumberBound<N> nb_max)__CURSOR_HERE__
to
   <A HREF="NumberInRange.html#NumberInRange(_4JDT_NumberBound, _4JDT_NumberBound)">NumberInRange</A>(nb_min, nb_max)
Above is a table of the features for each phrase. For example, CODE? means 'is the link generated by the phrase, as viewed in a web-browser, Java code? If yes, then
   - it can be surrounded by <CODE></CODE>,
   - it can have a prek link, as in
   <A HREF="NumberInRange.html">NumberInRange</A>.<A HREF="NumberInRange.html#NumberInRange(_4JDT_NumberBound, _4JDT_NumberBound)">NumberInRange</A>(nb_min, nb_max)
   - and it can also have type-abbreviations in the post-</A> (PAP) params:
   <A HREF="NumberInRange.html">NumberInRange</A>.<A HREF="NumberInRange.html#NumberInRange(_4JDT_NumberBound, _4JDT_NumberBound)">NumberInRange</A>(_4TV_NumberBound, _4TV_NumberBound)

(_4TV_[anything] is replaced with the abbreviation for [anything]. _4JDT_ is replaced with the fully-qualified package name for the following type.)
