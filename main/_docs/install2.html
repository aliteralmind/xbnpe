There are four XBNPE "global-configuration-variables" that must be set:

  • zROOT_DIR_XBNPE_PARENT
  • zWIN_DIR_DESKTOP     (optional)
  • zzTPTxtDefaultWordWrapOnOrES
  • zzTPTxtDefaultStripTrailingWSOnOrES
Note the phrase-file location and the PhraseExpress folder to which it needs to be imported. Set the values in the text-file and then import it to its required location in PhraseExpress.

Verify the directory values by copying the phrase-name (such as "zROOT_DIR_XBNPE_PARENT") and then, in the open-dialog's "File name" field, executing "cpcb", and pressing enter to confirm it.

Warning: It is strongly recommended that you edit these values in the on-disk text-files (plain-text phrase-files), and then import them. This is true when editing any values during this configuration process, and throughout much of the projects in XBN-PhraseExpress. Editing phrases directly in the PhraseExpress application, at they very least, during this installation processes, is strongly discouraged.

The final four steps. Brace yourself.

(I can do all of these steps for you!)

1. Setup TPKC-user-directory structures

Setup your TPKC-user-directory structures, based the TPKC-user-directory name you declared in the previous step. Note that much of these structures are already created.

2. Install all TextPad-bootstraps

Setup all the TextPad-bootstraps for the XKP-project and then setup all the TextPad-bootstraps for the TPKC-project.

The XKP and TPKC-projects are now fully-installed. Now it is time to teach XBN-PhraseExpress your custom TextPad key-commands.

3. Run the TPKC user-build-process

The final step is to run the TPKC user-build process, and import its output phrase-files. There are two parts, and they take a total of around an hour and twenty minutes. They are largely interaction-free, but you may not touch your computer while they are running.

DONE.

To verify that XBN-PhraseExpress is properly installed, execute these utilities in order, in a new TextPad document:

  1. testtpbsxkptpkc: Verifies all TextPad-bootstraps. more info
  2. vtpkcuds: Verifies your on-disk TPKC-user-directory. more info
  3. Assign all key-commands to a single TextPad command. For those with many custom key-commands, the true test is to reset all key-commands with
        TextPad > Configure > Preferences > Keyboard > Categories > "All" > [Reset-All button]
    and then re-assign them. Once complete, TextPad should behave exactly the same.

You may also wish to disable all default XBNPE key-commands at this time.
