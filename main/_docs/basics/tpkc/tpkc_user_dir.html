TPKC-user-directory and user-dir-name

The TPKC-user-directory is where your TextPad key-command setup is stored. This special directory

  • makes it possible to have multiple key-command "profiles" for TextPad,
  • gives you the ability to execute the TPKC-user-build for people on remote-computers--without disturbing your own setup, and
  • is the only folder that needs to be backed up, in order to save your unique XBN-PhraseExpress profile.
The key to the TPKC-user-directory is the TPKC-user-directory name.

[top]   TPKC remote-build installation: Step-by-step

After your remote build has been processed, you will receive your TPKC-user-directory in a compressed (zip) file. To install it:

  1. Unzip the archive into the TPKC-user-dir parent directory.
  2. Open PhraseExpress and select the folder 
    XBNPE: Parent folder >
   XBN-PhraseExpress >
      XBNPE-TPKC: TextPad built-in commands     (go)
  3. Click the key-command for importing: Ctrl+I
  4. Navigate into your TPKC-user-dir (what you just unzipped), then into the "output" directory, and select
        XBNPE-TPKC: User-spcfc phrases: [TPKC_USER_DIR_NAME].pxp
  5. Answer "no" to eliminating duplicate phrases.
  6. Enable the just-imported folder (right-click on it and check
        "Enable Autotext/Hotkeys"
    ), and disable all other
        "XBNPE-TPKC: User-spcfc phrases" folders

[top]   TPKC-user-dir: Remote build (request one now!)

To execute the TPKC user-build-process for someone else, without disturbing your own setup:

  1. Disable your own TPKC-user-directory (in PhraseExpress, right-click on the folder and check "Enable Autotext/Hotkeys"), but re-enable your main TextPad-bootstrap phrase-files.
  2. Setup and then enable the other person's TPKC-user-directory, but then disable their main tp-bootstrap phrase-files.
  3. Execute the TPKC user-build-process, importing all files to their TPKC-user-dir.
  4. Export their PhraseExpress TPKC-user-dir to their on-disk TPKC-user-dir (in the output directory)
  5. Zip up their on-disk TPKC-user-dir, and email it to them.
  6. Enable your entire TPKC-user-dir, and disable theirs.

There are three things that make it possible to execute the TPKC user-build-process for other users:

  • The TPKC-user-directory name is stored directly in its TPKC-user-directory. Simply enabling the desired directory (and disabling all others) activates its name.
  • All key-commands in your user-specific command-configuration (and the escaped TextPad-bootstrap phrases) are escaped.
  • TextPad-bootstraps are split into two files.

[top]   Your TPKC-user-directory name

Your TPKC-user-directory is uniquely identified by a name.

  • Phrase name: zzTpkcUserDirName
  • Example value: jsmith     (note "tp_frsh_instl" below)
  • Locations:
    • On-disk (phrase-file)
      • Before TPKC-user-directory installation: (as of 9/3/2013)
            {#insert zROOT_DIR_XBNPE_MAIN}textpad__tpkc\for_v_7_0_9\user_config\TPKC_USER_DIR_NAME\z_tpkc_user_directory_name.px.txt
      • After installation:
            {#insert zXPROOT_DIR_tpkcUsrCfgUserBase_M_tudn}z_tpkc_user_directory_name.px.txt
        This is the same directory path, except
            TPKC_USER_DIR_NAME
        has been changed to your TPKC-user-dir name.
    • PhraseExpress folder (to import to): The root folder of your TPKC-user-directory: 
          XBNPE: Parent folder >
       XBN-PhraseExpress >
(go)      XBNPE-TPKC: TextPad built-in commands >
(info)       XBNPE-TPKC: User-spcfc phrases: [TPKC_USER_DIR_NAME]

Requirements

  • Must be a valid Microsoft Windows directory name, and one that does not contain any special characters (such as spaces) that would require the entire path to be enclosed in double-quotes.
  • Must be relatively short (recommended no more than ten characters in length). The XBNPE (on-disk) directory structure is sufficiently deep, such that an excessively-long TPKC-user-dir name (or installation directory path) could violate the Windows maximum path-length.
  • Must be unique among all existing TPKC-user-directories, regardless which is currently active.
  • Only one TPKC-user-directory, and therefore only one TPKC-user-dir name, must be active at any one time.

tp_frsh_instl

If you have just installed TextPad for the first time, and have no custom key-commands (or just reset them with
    TextPad > Configure > Preferences > Keyboard > Categories > "All" > [Reset-All-button]
), use the "tp_frsh_instl" TPKC-user-directory name. This special TPKC-user-directory name prevents you from having to do much of the XBN-PhraseExpress installation process and all of the TPKC user-build-process (saving you more than an hour).

[top]   Setting up your TPKC-user-directory structures: Step-by-step

This section describes how to set up your TPKC-user-directory, both on-disk and in-PhraseExpress. It is expected that your TPKC-user-directory name has been both setup and imported.

(See: vtpkcuds)

Instructions for non-"tp_frsh_instl" users

Instructions for those with a TPKC-user-dir name equal to anything other than "tp_frsh_instl":

  1. In PhraseExpress, enable
        XBNPE-TPKC: User-spcfc phrases: [TPKC_USER_DIR_NAME]
    and disable all other
        XBNPE-TPKC: User-spcfc phrases: ...
    folders.
  2. Double-click on your folder (the one you just enabled) to expand it. Disable all folders named
        TPKC-user-build: Auto-generated -- ...
    except
        TPKC-user-build: Auto-generated -- LIVE
  3. Change
        [TPKC_USER_DIR_NAME]
    to your actual TPKC-user-directory name (the value of zzTpkcUserDirName).
  4. Close PhraseExpress.
  5. In Windows Explorer, navigate to the TPKC-user-dir parent-directory.
  6. You will be using the "TPKC_USER_DIR_NAME" directory, which contains blank phrase-templates, for both TextPad-bootstraps. Change the name of this folder to your TPKC-user-dir name.
  7. Back it up. Back it on up.

[top]   TPKC user-dir: On disk directory structure

The structure of the TPKC user-directory, as it exists on disk (in Windows):

  • {#insert zXPROOT_DIR_tpkcUsrCfgUserBase_M_tudn}
    • output_live\
      • nbs_command_phrases__bldproc14\     (info)
        • Output file: file.px.txt     (one per-category)
      • user_spcfc_cfg__bldproc10thr13\     (info)
        • indv_cat_lists__bldproc11\
        • raw_user_config__bldproc12\
        • Output file: file.px.txt     (one per-category)
      • Output file: tp_all_active_cmds__bldproc15__pcddmenu.px.txt     (info)
      • Output file: tp_all_active_cmds__pcddmenu__indented_DO_NOT_IMPORT.txt
    • output_test\     (...same structure...)
    • Input file: textpad_all_command_list.txt     (info)
    • Rqd file: tp_bootstraps_for_[xkp,tpkc]_project[_esc_tpk].px.txt     (info)
    • Rqd file: z_tpkc_user_directory_name.px.txt     (info)
    • Backup: XBNPE-TPKC- User-specific phrases- [TPKC_USER_DIR_NAME]     (info)

All "output_*" files are created by the TPKC user-build-process. All tp_bootstraps_* files and the textpad_all_command_list.txt are created by you.

Multiple "output_*" version directories

Each "output_*" directory represents the unique set of categories processed by the TPKC user-build-process. This is its "version". The currently-active version is defined by the TPKC master-configuration-variable
    zzTpkcBuildVersionName

                TPKC-user-build: Auto-generated -- LIVE >
(info)             Nbs TP-phrases [CALL THESE!] (AG:14) & has-a-kc qbl-s

(info)             z-DDMENU of all active TP-built-ins (AG:15)

The "LIVE" and "FGH-TEST" folders facilitate switching between TPKC-build versions. Exactly one version (either fgh or live) must be enabled at any one time. Which version should be enabled is determined by the TPKC master-config-variable
    zzTpkcBuildVersionName

[BACK]   TPKC user-dir: ...\output\user_spcfc_cfg__bldproc10thr13\file.px.txt

This directory contains your TPKC "user-specific command-configuration". It output by sub-build-proc-13, which is part of the TPKC user-build-process part 1/2.

Phrase and file locations:

  • On disk (phrase-file):
        {#insert zXPROOT_DIR_tpkcUsrCfgUserBase_M_tudn}output\user_spcfc_cfg__bldproc10thr13\file.px.txt
            (Replace "file" with any category in zzTpkcBuildVersionCatAbrvs)
  • [GO]   In PhraseExpress (where to import to): TPKC-user-directory

There is one phrase-file in this directory, for every category that

  • has at least one command,
  • whose nbs-kc-count is greater than zero
  • (whether or not it is a TextPad-bootstrap).

As a whole, these phrase-files are known as your "user-specific command-configuration" (or user-cmd-cfg). Whenever you change an nbs-kc in
    TextPad > Configure > Preferences > Keyboard
that change must be reflected in its user-cmd-cfg category-file. This is required in order for the assign-kc process to assign key-commands as you expect.

If the primary-kc is changed, then TPKC sub-build-processes-14 must be re-run. However, when it is known that only one-or-a-few categories have been changed, it is possible to execute build-proc-14 with category prompts.

If a command that never had a key-command now does (or you just deleted the final key-command from a no-longer-needed command), then it is strongly-recommended that sub-build-proc-15 also be re-run. This is in order to ensure the xh... phrase-creator ddmenu is accurate.

(When changing a TextPad-bootstrap, its main, escaped, and tp-key phrases must all be changed, in phrase-files unrelated to user-cmd-cfg.)

[BACK]   TPKC user-dir: ...\output\nbs_command_phrases__bldproc14\file.px.txt

This directory contains the nbs-phrases for all built-in TextPad commands to which you have assigned at least one key-command. These files are output by sub-build-proc-14, which is part of the TPKC user-build-process part 2/2.

Phrase and file locations:

  • [GO]   (phrase-file):
        {#insert zXPROOT_DIR_tpkcUsrCfgUserBase_M_tudn}output\nbs_command_phrases__bldproc14\file.px.txt
            (Replace "file" with any category in zzTpkcBuildVersionCatAbrvs)
  • [GO]   In PhraseExpress (where to import to): TPKC-user-directory

There is one phrase-file in this directory, for every category that

  • has at least one command,
  • whose nbs-kc-count is greater than zero, and
  • is not a TextPad-bootstrap.

These phrases are the final output of the TPKC project. When any phrase (outside of TPKC and XBN-PhraseExpress) needs to trigger a specific TextPad built-in command, these nbs-phrases are the ones they should call. The only exceptions are TextPad-bootstraps, which are not included in these phrase-files.

Phrase and file locations:

[BACK]   TPKC user-dir: ...\output\XBNPE-TPKC- User-specific phrases- [TPKC_USER_DIR_NAME]

This is the backup file of your PhraseExpress TPKC-user-folder. You are encouraged to back this folder up (by exporting it...select the folder, then type Ctrl+E) to the root of your on-disk TPKC-user-directory location after any changes are made.

The on-disk TPKC-user-directory should then be backed up by zipping it.
