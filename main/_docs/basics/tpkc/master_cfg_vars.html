[top]   TPKC master-cfg-var: zForceTestKeyCmd

The key-command used to "force" a TPKC nbs-command test. Not unlike TextPad-bootstraps, this also has a required corresponding "escaped" version:
    zForceTestKeyCmdEsc

For example, if zForceTestKeyCmd is equal to
    {#insert zASpOO}
then zForceTestKeyCmdEsc must be equal to
    \\\{#insert zASpOO\\\}

Requirements

  • This specific key-command (sequence of key-presses, independent of sleep) may not be assigned to any command, as configured in the TPKC-project. That is, it may not be equal to the key-command of any TextPad-bootstrap, nor the nbs-kc of any command (as specified in your user-specific command-configuration).
  • It is strongly-recommended that each key-press be an x-key-press.
  • When this key-command has multiple key-presses, they should be separated by some sleep (the default value is zTpkcMillsBtwMultiKeyKCs milliseconds).
  • It is not necessarily required to have sleep at they very end of this phrase, since it is always called in isolation (a message-box is popped up after each TPKC-nbs-test).

[top]   TPKC master-cfg-var: zzTpkcBuildVersionName

  • Requirements: Must be a valid (portion of a) directory-name, such that none of its characters (such as spaces) require the overall path to be surrounded by double-quotes.
  • Example values: "live" or "fgh"

Represents the currently-active TPKC-build "version", and is specifically the postfix of

  • the category-subset config-variable
        zzTpkcBuildVersionCatAbrvs_*
  • and the two on-disk output (auto-generated--"AG") phrase-file locations:
    • Sub-build-processes 01 through 09 (admin-only)
          {#insert zXPROOT_DIR_AG_tpkc_M_bvm}
    • Sub-build-processes 10 through 15 (user-only)
          {#insert zXPROOT_DIR_tpkcUsrCfgUserBase_M_tudn}output_{#insert zzTpkcBuildVersionName}\
      For example:
          C:\programming\phrase_express\xbnpe\main\textpad__tpkc\for_v_7_0_9\user_config\tp_frsh_instl\output_live\

In addition, the three PhraseExpress folder-locations (where the above phrase-files are imported), must each have a corresponding version-folder:

  • Sub-build-proces 01 through 09:
    • [GO]   the TPKC-user-directory, or
    • [GO]   the "other-AG-output" folder.
  • Sub-build-procs 10 through 15:
    • [GO]   The TPKC-user-menu
All locations, both on-disk and in PhraseExpress, must have the exact same versions.

(See: Switching between versions.)

[BACK]   TPKC-build-version: TPKC master-cfg-var: zzTpkcBuildVersionCatAbrvs_*

  • Requirements: Must be an exact subset of, or equal to, zTpkcAllCategoryAbrvLCsQbl.
  • Example value: [file][config][help]     (this is a good test-build)

The set of categories used by a specific TPKC-build-process. This is called its "version". The currently active version is determined by
    zzTpkcBuildVersionName
If zzTpkcBuildVersionName is "live", then
    zzTpkcBuildVersionCatAbrvs_live
is the currently active category-set. You can switch versions at any time.

Exactly one version must be called "live"--meaning there must be a phrase named
    zzTpkcBuildVersionCatAbrvs_live
and its contents must be equal to
    {#insert zTpkcAllCategoryAbrvLCsQbl}
All other versions, if any, are different flavors of testing-and-demonstration versions.

[BACK]   TPKC-build-version: Same versions in all locations.

[In PhraseExpress: user-dir, user-menu, other]

By default, there are two available TPKC-build versions. They are represented by the master-configuration-variables
    zzTpkcBuildVersionCatAbrvs_live     ("live" is a required version)
and
    zzTpkcBuildVersionCatAbrvs_fgh

zzTpkcBuildVersionCatAbrvs_live is equal to
    {#insert zTpkcAllCategoryAbrvLCsQbl}
and zzTpkcBuildVersionCatAbrvs_fgh is equal to
    [file][config][help]
(which is a good test build). The currently active version is determined by
    zzTpkcBuildVersionName

In all locations, both on-disk and in PhraseExpress, the exact same versions ("live" and "fgh") must be present. The current version only
must be enabled (right-click the folder and check
    "Enable Autotext/Hotkeys"

Using the above, they are represented by these on-disk directories:

  • ...\xbnpe\auto_generated\textpad__tpkc\output_fgh

and
  • ...\xbnpe\main\textpad__tpkc\for_v_7_0_9\user_config\tp_frsh_instl\output_fgh
  • ...\xbnpe\main\textpad__tpkc\for_v_7_0_9\user_config\tp_frsh_instl\output_live

And in each of these PhraseExpress folders (where each version-folder has exactly the same sub-directories):

[BACK]   TPKC-build-version: Switching between versions

To switch between TPKC-build versions:

Change the TPKC master-configuration-variable
    zzTpkcBuildVersionName
to the desired version. All available versions are determined by the existence of the master-config-vars named
    zzTpkcBuildVersionCatAbrvs_*
For example, set
    zzTpkcBuildVersionName
to "fgh", which refers to
    zzTpkcBuildVersionCatAbrvs_fgh
which is equal to
    [file][config][help]     (see: good test build)

(This section assumes that this version has indeed been built--both by admins [sub-build-procs 01 through 09] and users [10-15]--and that all on-disk and in-PhraseExpress directory structures exist! To create a new build-version, simply create the new zzTpkcBuildVersionCatAbrvs_* variable and all its corresponding structures, then run the entire build-process for it.)

This now refers to
    ...\xbnpe\auto_generated\textpad__tpkc\output_fgh
and
    ...\xbnpe\main\textpad__tpkc\for_v_7_0_9\user_config\tp_frsh_instl\output_fgh

In PhraseExpress the following locations must have their corresponding version folders enabled:

  • TPKC-user-menu: Enable and display both FGH-TEST folders:
    • To enable: right-click on each folder and check
          "Enable Autotext/Hotkeys"
      To display: Check
          "Display in pop-up menu"
    • Disable and hide all other version-folders. The user-menu is the only location where items must be hidden or displayed.
  • In both the TPKC-user-directory and the "other" folders: Enable the FGH-TEST folder, and disable all other version-folders.

[top]   TPKC master-cfg-var: zTpkcMaxNbsKCsPerCmd

Requirements: Must be an integer greater than zero.

The maximum number of key-commands that any one built-in TextPad command may have, after all bootstrap key-commands have been deleted. This is validated by the TPKC-build-process (sub-build-proc-10), which specifically analyzes your TextPad all-command-list. (Note that a larger number will make the TPKC-build process slower. The default is 5.)

Convenience utility (xhtk): To sort of validate the number of nbs-kcs in your tp-all-cmd-list, open the file, and within it, execute
    tpaclwarnkccount
Since your tp-all-cmd-list contains bootstrap-kcs, it not possible to truly validate this. If this utility gives you a "SUCCESS" message, then you can be assured that it is indeed good. However, it is possible for this utility to give you a "false negative", because its counting includes your bootstrap-kcs, and it shouldn't (it can't know).

The nbs-kc count validation at the end of sub-build-proc-10 is accurate.

[top]   TPKC master-cfg-var: zTpkcMillsBtwMultiKeyKCs

  • Requirement: Must be an integer greater than zero (a number valid for {#sleep ...})
  • See: The global-config-variable zTpkc[abrv]DfltPostKcSlpMils

The number of milliseconds to sleep between multi-key key commands. For example, if your TextPad all-command-list has this command-line:
    FileCloseAll            Ctrl+F12, Ctrl+F9
and zTpkcMillsBtwMultiKeyKCs is equal to 25, then the TPKC-build-process translates it into this nbs-phrase:

zTPFileCloseAll
   {#insert zCpF12}{#sleep 25}{#insert zCpF9}

You are free to change this millisecond-value once the nbs-phrase is created. This is also the recommended value for between key-presses in the main and escaped phrases in your TextPad-bootstraps .

(The phrase-name for the command FileCloseAll is configured to be zTPFileCloseAll.)
