XBN-PhraseExpress

[start, basics, devpex, tasks, generic, future]

XBN-PhraseExpress is a large set of generically-useful Microsoft Windows scripting tools, buith in PhraseExpress* and TextPad as its foundation. XBNPE is significantly for creating phrases in TextPad, and therefore many phrases are logically only-callable in a TextPad document. There are, however, many useful features for those developing phrases in any environment, including the PhraseExpress application itself.

There is a significant setup necessary in order for XBN-PhraseExpress and TextPad to communicate properly, but it is largely a one-time-only process.

[BACK]   XBNPE: Getting started: How to "execute" a phrase

To execute a phrase, do one of the following

  • Type it's key-command ("hotkey")
  • Type its name into a TextPad document, followed by the accent key ('`'--on my keyboard it's on the same key as the tilde: '~'). The accent is the default trigger-postfix in XBN-PhraseExpress
  • Copy its name, then type
        cpb`

XBNPE: Live example code

All example code throughout XBN-PhraseExpress documentation is "live"--it can be executed in a new TextPad document. Adding "wsc" to the end of any example-code phrase-name, also executes necessary setup and cleanup. For example: "snr06findselxtoz", and "snr06findselxtozwsc".

This is made possible by the as-of-yet-unfinished XBNPE-Example-code framework (XCF-project).

Note that example code is presented in free-spacing mode (inspired by Mastering Regular Expressions author Jeffrey E. F. Friedl). You can also copy the example code yourself, directly off of a documentation webpage. To make it usable:

  1. Select and copy it with your mouse, and paste it into a new text document (save it as a phrase-file named "something.px.txt")
  2. Eliminate all line-breaks and indentation, except for the line-break between the phrase-name and body. Spaces at the end of lines must be preserved (there are hidden non-breaking spaces--"&nbsp;" in the HTML source code, at the end of source lines where necessary).
  3. Since the phrase already exists, its name must be changed. Consider adding a unique postfix, such as your initials ("snr06findselxtoz" --> "snr06findselxtozjf")
  4. This is now an uncompressed phrase. Compress it with "pc2".
  5. Import something.px.txt.

Also note all example code can be slowed.
