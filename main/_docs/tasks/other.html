[TOP]   Reading Inline-Configuration-Variables

In TextPad, execute 'icvdemo'. This creates four new demo variables in a new document, and then reads them in, one by one. Each variable can be defined as required or optional (meaning must have at least one character following the '='). Required variables may also have default values, used only when no value (no characters) is provided.

Syntax

  • Value required: zSetCBToNamedInlineCVWRqdValGTNxtLn_M_vnm
  • Value required w/ no-value-provided default: zSetCBToNamedOptInlineCfgValESDfltIsESGTNxtLn_M_vnmDflt
  • Value optional: zSetCBToNamedOptInlineCfgValESDfltIsESGTNxtLn_M_vnmDflt
  • Other:
    • zSetCB_TO_dflt
    • zvES_VAL_DEFAULT_stES
    • zvOUT_DIR_stRqdInlineCV
    • zvOUTPUT_FILE_NAME_stOptInlineCVOrThisFileNm
    • zzErrDlg[MissingRqdVal,MissingICV,BadICVValue]_M_cvn

Example inline-config-variable from icvdemo[go] (Phrase-creators: xhoi)

  • Source-code: {#insert zROOT_DIR_XBNPE_MAIN}other\inline_config_vars\icv_demo_primary.px.txt

In TextPad, execute icvdemo to see this in action.

zzIcvSetRqd_emailTo
(*Define the config-var name, as required by zSetCBToNamedInlineCVWRqdValGTNxtLn_M_vnm:*)
{#SETPHRASE -description zvCFG_VAR_NAME -content __EMAIL_ADDR_TO__ -autotext zvCFG_VAR_NAME -folder ctvv_folder}

(*Read this REQUIRED variable from the text file--the current line--and set it into the clipboard:*)
{#insert zSetCBToNamedInlineCVWRqdValGTNxtLn_M_vnm}

(*Transfer the value from the clipboard to the required Pex variable:*)
{#SETPHRASE -description zvEMAIL_TO -content {#INSERTCLIPBOARD} -autotext zvEMAIL_TO -folder ctvv_folder}

(*Additional error checking: The email must have an '@'. If it does, do nothing (pause for 1 millisecond). If it doesn't present a generic error-dialog:*)
{#insert {#COND -if {#POS {#insert zvEMAIL_TO} -substr @} > 0 -then z1slp -else zzErrDlgBadICVValue_M_cvn}}
